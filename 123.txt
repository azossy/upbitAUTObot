■ 기록 규칙 (형식)
--------------------------------------------------------------------------------
- 작업/검증이 **끝나면** 해당 블록 맨 위에 반드시 **[완료!]** 와 완료 일자(YYYY-MM-DD)를 적어 주세요.
  예: [완료!] 2025-03-02 — 상용 프로그램 1차 개발
- 진행 중인 작업은 **[진행중]** 으로 표시. 완료 시 [진행중]을 [완료!] + 날짜로 바꿔 주세요.
- "■ 작업 완료 현황" 섹션에서 전체 진행을 한눈에 확인할 수 있으니, 완료 시 해당 줄도 갱신해 주세요.
- **[개발관·검증관] 대기 중 동작**: 작업이 끝나서 할 일이 없을 때는 **30초마다** 123.txt 파일을 계속 확인해 주세요. 새로운 작업 지시나 검증 요청이 왔는지 확인하는 것이 필요합니다.

■ 에이전트 역할 및 이름 (123.txt 사용 시 아래 이름·역할로 협업)
--------------------------------------------------------------------------------
  [기획관] P에이전트
    - 역할: 기획·우선순위 정리, 다음 개발 단계 제안, UI/UX 기획(One UI 가이드 기준), 개발관·검증관에게 지시·안내 전달
    - 담당: "기획관 → 개발관·검증관 지시·안내" 섹션 작성, docs/기획_다음단계.md·docs/UI_UX_가이드_적용.md 갱신
    - 123.txt 확인 시: 진행 상황 반영 후 기획서 업데이트, 필요 시 지시·안내·작업 지시에 UI/UX 요구사항 반영

  [개발관] 코딩 에이전트
    - 역할: "작업 지시" 내용에 따른 코딩 수행, 결과를 "최근 작업 내역"에 기록. **검증 후 오류·수정사항**이 "코드 검증 결과 및 작업지시서"의 "■ 작업지시서"에 있으면 **해당 항목대로 수정·보완 코딩** 수행.
    - 담당: 123.txt 확인 → 작업 지시 있으면 코딩 → 최근 작업 내역 업데이트. **작업 완료 시** 해당 작업 블록에 **[완료!] YYYY-MM-DD** 표기, "■ 작업 완료 현황" 테이블 갱신.
    - **대기 루프 (반복)**: (1) 대기 중 → (2) **30초마다** 123.txt 확인 → (3) 새 작업 발견 시 작업 시작 → (4) 작업 끝 → (5) 결과 보고서를 123.txt에 업데이트(최근 작업 내역·완료 표기·검증 요청 등) → (6) 대기 중으로 복귀 → (2)부터 반복.
    - 30초마다 확인하려면: 이 채팅에 "123 확인" 또는 아무 메시지를 **30초마다** 보내거나, 터미널에서 `python check_123_interval.py` 실행해 두면 30초마다 확인 안내가 출력됨(해당 시점에 채팅에 "123 확인" 입력하면 개발관이 123 확인·작업 수행).
    - 코딩 완료 후: 필요 시 "검증관 할당 작업"에 검증 요청 한 줄 기재 (검증관이 확인하도록)

  [검증관] 테스트 에이전트 (코드체크·프로그램 테스트)
    - 역할: 개발관의 코딩 결과 검증. **논리 오류가 없는지, 기획·작업 지시대로 정확히 구현되었는지** 확인하고, 품질·보완 사항 점검 후 "코드 검증 결과 및 작업지시서" 갱신. **오류·수정사항이 있으면** "■ 작업지시서"에 번호로 적어 **개발관이 수정 작업**하도록 전달.
    - 담당: "검증관 할당 작업"에 검증 요청이 올라오면 검증 수행 → 검증 결과 섹션 작성 → 할당 섹션에 **[완료!] YYYY-MM-DD** 표기, "■ 작업 완료 현황" 테이블 갱신
    - **대기 중**: 검증 요청이 없을 때는 **30초마다** 123.txt를 확인해 새 검증 요청이 왔는지 계속 확인해 주세요.
    - **기획관 지시 준수**: 기획관(P에이전트)이 "기획관 → 개발관·검증관 지시·안내" 섹션에 적은 요구사항을 따름. 검증 시 상용 프로그램 수준·논리 오류·예외처리·보안·가독성·사용성·에러 메시지 명확성·UI/UX(One UI 기준) 반영.
    - 자동 모니터링: 터미널에서 python monitor_123.py 실행해 두면, 123.txt를 주기적으로 확인하다가 검증 요청이 올라오면 자동으로 린트/검증 후 본 문서 갱신. (사용자님이 매번 "123 확인해줘" 하지 않아도 됨)

  1. 대기 중
  2. 30초마다 123.txt 확인
  3. 새 작업 발견 → 작업 시작
  4. 작업 끝
  5. 결과 보고서 123에 업데이트 (최근 작업 내역, [완료!] 표기, 검증 요청 등)
  6. 대기 중으로 복귀 → 2번부터 반복

※ 30초마다 확인: 채팅에 "123 확인" 등을 30초마다 보내거나, 터미널에서 python check_123_interval.py 실행.

   - 검증관: 5회 실패 후 403+remaining_minutes 반환, 앱에서 남은 분 표시 확인.
