## 2. 시장 국면 판단

시장 국면 판단은 8개 지표를 점수화하여 합산하는 방식으로 결정됩니다.

### 2.1 8개 지표 점수 체계

```
지표              상승 신호 (+1)           중립 (0)         하락 신호 (-1)
────────────────  ───────────────────────  ───────────────  ────────────────────────
ADX               30 이상                  20 ~ 30          20 이하
BTC EMA 배열      정배열 + 기울기 양수     뒤엉킴           역배열 + 기울기 음수
볼린저 밴드 폭    평균 대비 130% 이상      70~130%          70% 이하
펀딩비            +0.01% ~ +0.05%          ±0.01% 이내      0% 이하 (음수)
OI 변화율 (1h)    +3% 이상 증가            ±3% 이내         -3% 이상 감소
롱/숏 비율        55~65% 롱                50~55% 롱        45% 이하 롱
온체인 점수       +0.5 이상                -0.5 ~ +0.5      -0.5 이하
F&G Index         55 ~ 75                  45 ~ 55          25 이하 or 75 이상
```

### ✅ [개선 5] F&G Index 급변 구간 예외처리

```
# ─────────────────────────────────────────────────────────────────
# [개선 이유]
#   F&G Index는 하루 1회만 업데이트됨.
#   급격한 시장 변화 시 전날 지수가 그대로 유지되어 점수 왜곡 발생.
#   예) 장중 BTC 급락 -20% 발생 시에도 전날 탐욕 지수(70)가 유지됨.
#
# [개선 내용]
#   - 정상 구간: F&G Index 기존 가중치 1점 적용
#   - 급변 감지 구간: F&G Index 점수를 0점(중립)으로 고정하고,
#     나머지 7개 지표로만 국면 판단
#   - 급변 감지 조건 (아래 중 하나 충족 시):
#       ① BTC 1시간봉 변동폭 ±5% 초과
#       ② OI 1시간 변화율 ±10% 초과
#       ③ 대규모 청산 이벤트 발생 (청산 규모 1억 달러 이상)
#   - 급변 감지 상태 해제: 4시간 연속 정상 구간 유지 시 자동 해제
# ─────────────────────────────────────────────────────────────────

급변 감지 플래그 = False  # 초기값

if BTC_1h_변동폭 >= 5% OR OI_1h_변화율 >= 10% OR 대규모청산_발생:
    급변 감지 플래그 = True
    F&G 점수 = 0   # 중립 고정, 나머지 7개 지표로만 판단

if 급변 감지 플래그 == True AND 4시간 연속 정상:
    급변 감지 플래그 = False
    F&G 점수 = 정상 계산값 복귀
```

### ✅ [개선 6] 섹터 로테이션 교체 방식 개선 (부분 교체 도입)

```
# ─────────────────────────────────────────────────────────────────
# [개선 이유]
#   v1.0에서는 4가지 조건 모두 충족 시에만 교체가 가능했음.
#   특히 "현재 보유 코인이 수익 중이면 교체 보류" 조건은
#   트렌드 전환 초기에 기존 코인을 계속 보유하게 만들어
#   수익이 손실로 전환되는 상황을 방치할 수 있었음.
#
# [개선 내용: 2단계 교체 방식]
#
# 1단계. 부분 교체 트리거 (조건 3개만 충족해도 실행 가능)
#   - 새 섹터가 2시간 연속 상위권 유지
#   - 신규 섹터 스코어가 현재 섹터보다 20% 이상 높음
#   - 마지막 교체 후 최소 4시간 경과 (쿨타임)
#   → 위 3개 충족 시: 현재 보유 코인의 50%만 청산 후 신규 코인 편입
#     (수익 중인 코인도 절반은 계속 보유 → 추가 수익 가능)
#
# 2단계. 전량 교체 트리거 (조건 4개 모두 충족 시)
#   - 위 3개 조건 + 현재 보유 코인 손익 -1% 이하
#   → 기존 코인 전량 청산 후 신규 코인으로 교체
#
# [강제 교체 최소화 원칙은 유지]
#   - 손절/청산으로 자연스럽게 생긴 슬롯에 새 코인 편입 우선
#   - 잦은 교체는 수수료(0.05% 양방향) + 슬리피지 누적으로
#     수익을 갉아먹으므로 부분 교체로 중간 타협점 확보
# ─────────────────────────────────────────────────────────────────

교체 방식 의사결정 트리:
  조건 3개 충족 AND 현재 보유 코인 수익 중
    → 보유 코인의 50% 청산 + 신규 코인 50% 편입 (부분 교체)

  조건 3개 충족 AND 현재 보유 코인 손익 -1% 이하
    → 전량 청산 + 신규 코인 전량 편입 (완전 교체)

  조건 미충족
    → 현재 포지션 유지
```

### ✅ [개선 1] EMA 크로스 후행성 보완 — 시장 모드별 유동적 대기시간

```
# ─────────────────────────────────────────────────────────────────
# [개선 이유]
#   v1.0에서는 눌림목 최대 대기시간이 모든 상황에서 고정 4시간이었음.

#   강한 추세장에서 진입 기회를 완전히 놓치는 문제 발생.
#
# [개선 내용: 시장 점수에 따른 3단계 유동적 대기시간]
#
# 시장 점수 +8 (강한 상승 확신):
#   - 눌림목 대기시간: 2시간 (단축)
#   - 눌림목 미발생 시: 조건 완화 진입 허용
#     → RSI 60 이하 + 거래량 평균의 80% 이상이면 즉시 진입
#
# 시장 점수 +6 ~ +7 (일반 상승):
#   - 눌림목 대기시간: 4시간 (기존 유지)
#   - 눌림목 미발생 시: 신호 무효 처리 (기존과 동일)
#
# 시장 점수 +4 ~ +5 (약한 상승):
#   - 눌림목 대기시간: 6시간 (연장)
#   - 눌림목 조건 강화: RSI 45 이하 + 거래량 평균 이하 (기존보다 엄격)
#
# [강한 추세장 즉시 진입 조건 (시장 점수 +8 전용)]
#   ① ADX 35 이상 (강한 추세 확인)
#   ② BTC 1분봉 3개 연속 양봉
#   ③ 코인 자체 4시간봉 ADX 30 이상
#   → 위 3개 모두 충족 시 눌림목 대기 생략, 시장가 즉시 진입
# ─────────────────────────────────────────────────────────────────

시장 점수별 눌림목 대기시간:
  +8점   → 최대 2시간 대기 (강한 추세장, 조건 완화 즉시 진입 가능)
  +6~+7  → 최대 4시간 대기 (일반 상승장, 기존 기준 유지)
  +4~+5  → 최대 6시간 대기 (약한 상승장, 조건 강화)
```

## 6. 청산 로직

**청산 방식: 분할 청산 — 수익 구간을 나눠 단계적으로 청산하여 수익 확보와 추세 추종을 동시에 달성**

### ✅ [개선 2] 수수료 + 슬리피지 반영 실제 손익비

```
# ─────────────────────────────────────────────────────────────────
# [개선 이유]
#   v1.0은 손절 -2.5%, 익절 +5/+10/+15%를 명목 수치로만 표기함.
#   실전에서는 수수료 + 슬리피지가 더해져 실제 손익이 달라짐.
#   특히 분할 진입(3회) + 분할 청산(3회) = 최소 6회 거래 발생.
#   수수료 누적이 예상보다 크며, 이를 명확히 인지해야 함.
#
# [수수료 계산 기준]
#   업비트 현물 거래 수수료: 0.05% (매수 + 매도 = 0.10% 왕복)
#   슬리피지 예상치: 시장가 주문 0.10% ~ 0.20% (유동성 양호 코인 기준)
#   알트코인 급변 시 슬리피지: 최대 0.50% ~ 0.70%
#
# [실제 손익 계산표]
#
#   거래 횟수별 수수료 누적:
#     분할 진입 3회 × 0.05% = 0.15%
#     분할 청산 3회 × 0.05% = 0.15%
#     총 수수료 합계          = 0.30%
#
#   슬리피지 예상 합산 (시장가 주문 기준):
#     1차 매수(시장가) + 손절(시장가) + 3차 익절(시장가) = 약 0.30%~0.60%
#
#   ─────────────────────────────────────────────────────
#   시나리오별 실제 손익 (명목 기준 → 실제 기준):
#
#   ① 1차 익절 +5% 도달 (30% 청산):
#      명목 수익 +5.0% → 실제 수익 약 +4.6% ~ +4.7%
#
#   ② 2차 익절 +10% 도달 (30% 추가 청산):
#      명목 수익 +10.0% → 실제 수익 약 +9.6% ~ +9.7%
#
#   ③ 3차 익절 +15% 도달 (나머지 40% 청산):
#      명목 수익 +15.0% → 실제 수익 약 +14.5% ~ +14.6%
#
#   ④ 손절 -2.5% 발동:
#      명목 손실 -2.5% → 실제 손실 약 -2.8% ~ -3.1%
#      (슬리피지 높으면 -3.1%까지 확대 가능)
#   ─────────────────────────────────────────────────────
#
# [실제 손익비 (Risk-Reward Ratio)]
#   이론상 R:R = 1 : 2 (손절 -2.5% vs 1차 익절 +5%)
#   실제  R:R = 1 : 1.5 ~ 1.7 (수수료+슬리피지 포함 시)
#
# [백테스팅 필수 항목]
#   반드시 수수료(0.30%) + 슬리피지(0.30%~0.60%)를 포함한
#   실제 순수익 기준으로 전략 성과를 평가할 것.
#   명목 수익 기준 백테스팅은 실전과 괴리가 크게 발생함.
# ─────────────────────────────────────────────────────────────────

[익절 시 실제 수익 목표 기준 (내부 추적용)]
  1차 익절: 명목 +5.0%  →  실제 목표 +4.6% 이상
  2차 익절: 명목 +10.0% →  실제 목표 +9.6% 이상
  3차 익절: 명목 +15.0% →  실제 목표 +14.5% 이상

[손절 시 실제 손실 허용 범위]
  명목 -2.5% →  실제 예상 손실 -2.8% ~ -3.1%
  포트폴리오 손실 계산 시 실제 손실 기준(-3.1%)으로 집계
```

### 6.1 익절 — 분할 청산

```
구분        조건           청산 비율       주문 방식
──────      ──────────     ───────────     ─────────────────────────────
1차 익절    +5% 도달       보유량의 30%    지정가 청산
```


### 6.3 시간 손절 (개선판)

```
# ─────────────────────────────────────────────────────────────────
# [개선 이유]
#   v1.0 시간 손절은 "12시간 경과 + 수익률 -0.5%~+1.5%"이면 무조건 청산.
#   그러나 강한 상승장에서는 단순히 눌림목/횡보 중인 코인을
#   12시간 후 조기 청산하는 문제가 발생할 수 있음.
#   국면을 재확인하지 않으면 수익 기회를 성급하게 차단하게 됨.
#
# [개선 내용: 시간 손절 전 국면 재확인 조건 추가]
#
# 기존 조건:
#   진입 후 12시간 경과 + 수익률 -0.5% ~ +1.5% 구간 → 전량 청산
#
# 개선 조건 (아래 2가지 모두 충족 시에만 청산):
#   ① 진입 후 12시간 경과 + 수익률 -0.5% ~ +1.5% 구간 (기존 유지)
#   ② 현재 시장 국면 점수가 진입 시점 대비 -2점 이상 하락
#      OR 코인의 ADX가 20 이하로 하락 (추세 소멸 확인)
#
# [판단 분기]
#   12시간 경과 + 횡보 구간 BUT 국면 점수 유지 AND ADX 25 이상
#     → 청산 보류, 4시간 연장 대기 후 재판단 (최대 1회 연장)
#
#   12시간 경과 + 횡보 구간 AND 국면 점수 하락 OR ADX 20 이하
#     → 전량 청산 → 기회비용 제거 → 더 강한 신호 코인으로 교체
# ─────────────────────────────────────────────────────────────────

[시간 손절 의사결정 흐름]
  12시간 경과 + 수익률 -0.5%~+1.5%?
    ↓ YES
  국면 점수 하락(-2) 또는 ADX ≤ 20?
    ↓ YES                    ↓ NO
  전량 청산 실행           4시간 연장 대기 (최대 1회)
                               ↓
                          재판단 후 동일 조건 충족 시 청산
```

### ✅ [개선 3] Blockchain.com 비공식 API 폴백 전략

```
# ─────────────────────────────────────────────────────────────────
# [개선 이유]
#   v1.0에서 Blockchain.com API를 "무제한(비공식)"으로 표기.
#   비공식 API 엔드포인트는 사전 예고 없이 차단되거나 변경될 수 있음.
#   BTC 온체인 데이터는 시스템의 핵심 입력값이므로,
#   단일 소스 의존은 시스템 안정성에 심각한 리스크.
#
# [개선 내용: 3단계 폴백 전략]
#
# 1순위 (메인): Blockchain.com API
#   - 현재 사용 중인 엔드포인트 유지
#   - 3회 연속 오류 발생 시 자동으로 2순위로 전환
#
# 2순위 (폴백 1): CryptoQuant 무료 티어
#   - 사이트: cryptoquant.com
#   - 제공 데이터: BTC Exchange Reserve, Inflow, Outflow
#   - 무료 제한: 시간당 30회 호출 (10분 주기로 사용 시 충분)
#   - 가입 후 API 키 발급 필요
#
# 3순위 (폴백 2): Glassnode 무료 티어
#   - 사이트: glassnode.com
#   - 제공 데이터: BTC Exchange Net Position Change (일별 무료)
#   - 무료 제한: 일별 데이터만 제공 (실시간 불가)
#   - 긴급 시 보조 참고용으로만 사용
#
# [자동 폴백 로직]
#   if Blockchain.com 3회 연속 오류:
#       온체인 소스 = CryptoQuant   # 2순위로 자동 전환
#       텔레그램 알림 발송 ("BTC 온체인 소스 전환: CryptoQuant")
#
#   if CryptoQuant도 오류:
#       온체인 BTC 점수 = 0 (중립)  # 데이터 없으면 중립 처리
#       텔레그램 알림 발송 ("BTC 온체인 데이터 수집 실패, 중립 처리")
# ─────────────────────────────────────────────────────────────────

BTC 온체인 API 우선순위:
  1순위: Blockchain.com   (비공식, 현행 유지)
  2순위: CryptoQuant      (공식 무료 티어, 폴백)
  3순위: 중립값 처리      (모든 소스 실패 시)
```

### ✅ [개선 4] X(Twitter) API 한도 최적화 전략

```
# ─────────────────────────────────────────────────────────────────
# [개선 이유]
#   X API 무료 티어는 월 50만 트윗 읽기로 제한됨.

#   - 월 사용량 90% 도달 시: X API 수집 중단, Reddit으로 완전 대체

#
# [사전 테스트 권장]
#   실제 배포 전 1주일간 수집 테스트 실시.
#   대상 코인 3개 × 30분 주기 × 24시간 × 7일 = 예상 호출 횟수 확인.
#   월간 환산 시 50만 한도 내 수용 가능 여부를 확인 후 조정.
# ─────────────────────────────────────────────────────────────────

X API 수집 전략 요약:
  수집 대상  : 보유 코인 1~3위만 (전체 대신 상위 코인 집중)
```

## 11. 백테스팅 계획 (개선판)

```
# ─────────────────────────────────────────────────────────────────
# [추가 이유]
#   v1.0은 "백테스팅 → 페이퍼 트레이딩 → 소액 실거래" 순서만 언급.

#   백테스팅 결과가 실전과 크게 괴리될 수 있음.
# ─────────────────────────────────────────────────────────────────

[백테스팅 필수 원칙]

  1. 수수료 + 슬리피지 반드시 포함
     - 수수료: 0.10% (왕복)
     - 슬리피지: 시장가 주문 0.10%~0.20%, 급변 시 0.50% 적용

  2. 기간별 분리 검증 (시장 환경별 전략 성과 확인)
     - 2021년 (강세장 대표 구간)
     - 2022년 (하락장 대표 구간)
     - 2023~2024년 (횡보/회복 구간)

  3. 성과 평가 기준 (모두 충족 시 전략 유효)
     - Sharpe Ratio: 1.5 이상
     - Max Drawdown (MDD): 20% 이하
     - 승률: 52% 이상
     - 월평균 수익률: +3% 이상 (순수익 기준)

  4. 멀티 팩터 가중치 그리드 서치 최적화
     - 5% 단위로 조합 탐색 (총합 100% 조건 유지)
     - 시장 모드별 최적 가중치 세트 도출

  5. 재최적화 주기
     - 실전 투입 후 매 분기(3개월) 1회 파라미터 재검토
     - 연속 손절 5회 이상 또는 월 손실 -7% 이상 시 즉시 재검토

[단계별 실전 전환]
  STEP 1: 백테스팅 (최소 2년치 데이터, 위 기준 모두 통과 시)
  STEP 2: 페이퍼 트레이딩 (실제 거래 없이 3개월 시뮬레이션)

  STEP 4: 정상 운용 (결과 검증 후 본 설계 비율대로 운용)
```
