## 1. 대표 대형코인 온체인 데이터

업비트 현물 거래 코인 중 **시가총액·유동성이 큰 대표 코인**(BTC, ETH, XRP, SOL 등)의 온체인 데이터를 보면, **거래소 유입·유출·대형 자금 이동**을 파악해 진입·청산·비중 판단에 보조로 쓸 수 있음.

### 1.1 참조할 온체인 항목 (요약)

| 항목 | 의미 | 활용 (요약) |
|------|------|-------------|
| **거래소 예비금(Reserve) / 유입(Inflow) / 유출(Outflow)** | 거래소 지갑으로 들어오고 나가는 양 | 유입↑ → 매도 압력 가능성; 유출↑ → 보유·매수 심리 보조 지표. 설계서 8개 지표 중 "온체인 점수"에 반영. |
| **온체인 거래량** | 블록체인 상 실제 이체량(거래소 거래량과 구분) | 급증 시 큰 움직임 전조; 평소 대비 비율로 이상 징후 감지. |
| **대형 자금 이동(웨일/지갑 이동)** | 일정 규모 이상의 단일 또는 소수 트랜잭션 | 거래소 입금/출금 대량 발생 시 단기 변동성·방향성 참고; "스마트머니" 동향 보조. |

위 항목을 **대표 코인별**로 어떤 소스에서 가져올지 아래에 정리함.

### 1.2 코인별 온체인 데이터 소스 및 수집 항목

#### BTC (비트코인)

| 소스 | 수집 항목 | 비고 |
|------|-----------|------|
| **Blockchain.com** (또는 폴백: CryptoQuant) | Exchange Reserve, **Inflow**, **Outflow** | 설계서 10.2·개선3. 10분 주기. 폴백 순서: Blockchain.com → CryptoQuant → 중립 처리. |
| **CryptoQuant** (폴백/보조) | Exchange Inflow/Outflow, Reserve, Miner Position 등 | 무료 티어 한도 내에서 거래소 유입·유출·예비금. |
| **(선택) 대형 이동** | 100 BTC 이상 등 단위 거래 탐지 | Whale Alert 등 서비스 참고 가능; API 연동 시 별도 검토. |

- **활용**: 유출 증가 + 펀딩비 등 다른 지표와 맞물리면 상승 가능성 보조; 유입 급증 시 매도 압력·변동성 주의. 설계서 "온체인 점수"에 반영.

#### ETH (이더리움)

| 소스 | 수집 항목 | 비고 |
|------|-----------|------|
| **Etherscan** (공식 API) | **거래소/컨트랙트 지갑 잔액 변화**, 대형 트랜잭션(예: 1,000 ETH 이상) | 10분 주기. 설계서 1.1·10.2. |
| **(추가) 거래량** | 일일/시간별 체인 상 전송량 | Etherscan 또는 서드파티 요약 API로 요약치만 사용 가능. |

- **활용**: 거래소 지갑 잔액 증가 = 유입(매도 압력 가능), 감소 = 유출(보유·매수 심리). 대형 코인 이동 시 단기 변동성·방향 참고.

#### XRP (리플)

| 소스 | 수집 항목 | 비고 |
|------|-----------|------|
| **XRPL.org** (공식) | **거래소 계정 잔액**, 대량 이동 | 5분 주기. 설계서 1.1·10.2. |

- **활용**: 거래소 계정 잔액·이동량 추이로 유입/유출·대형 이동 감지.

#### SOL (솔라나)

| 소스 | 수집 항목 | 비고 |
|------|-----------|------|
| **Solscan** (public-api.solscan.io) | **거래소/대형 지갑 잔액 변화** | 5분 주기. 설계서 1.1·10.2. |

- **활용**: ETH와 유사하게 거래소·대형 지갑 기준 유입/유출·이상 이동 참고.

### 1.3 “큰 자금이 지갑에서 이동 중”일 때의 활용

- **거래소 유입(Inflow) 급증**: 보통 매도 압력·변동성 증가 가능성 → 진입 보수화, 비중 축소, 또는 청산 조건 민감도 상향 검토.
- **거래소 유출(Outflow) 급증**: 보통 보유·매수 심리 보조 → 기존 진입·비중 로직 보조 확인용; 단독 신호로 과매수하지 않음.
- **특정 규모 이상 단일 트랜잭션**(웨일 이동): 거래소 입·출금이면 위와 같이 해석; 비거래소 간 이동은 시장 직접 영향은 제한적이므로 참고만.

설계서의 **진입 금지 조건** “④ 온체인 점수 -0.5 이하 (매도 압력 강함)” 및 **재진입 조건** “온체인 Outflow 증가 + …”와 연계해, 위 온체인 항목을 **점수화·플래그**로 넣으면 됨.

### 1.4 구현 시 체크리스트 (온체인)

- [ ] BTC: Reserve/Inflow/Outflow 수집 (Blockchain.com → CryptoQuant 폴백), 10분 주기.
- [ ] ETH: Etherscan으로 거래소·대형 지갑 잔액 변화·(가능 시) 대형 트랜잭션, 10분 주기.
- [ ] XRP: XRPL.org로 거래소 계정 잔액·이동, 5분 주기.
- [ ] SOL: Solscan으로 거래소·대형 지갑 잔액 변화, 5분 주기.
- [ ] (선택) 온체인 거래량 급증 비율, 대형 이동(웨일) 플래그를 “온체인 점수” 또는 별도 보조 플래그로 반영.
- [ ] 설계서 “온체인 점수 -0.5 이하” 진입 금지, “Outflow 증가 + …” 재진입 조건과 연동.

## 2. 신뢰할 만한 코인 정보 사이트 참조

자동매매는 **로직 + 거래소 시세 + 온체인**을 1차로 두고, **신뢰할 만한 정보 사이트**는  
- 자동 수집 가능한 지표(API 등)가 있으면 **점수·필터에 보조 반영**  
- API가 없거나 수동 참고용이면 **운영자가 판단 시 참고**  
하도록 구분해 두는 것이 좋음.

### 2.1 시세·시가총액·기본 정보 (공식·업계 표준에 가까운 곳)

| 사이트 | 제공 정보 | 연동 방식 제안 | 비고 |
|--------|-----------|----------------|------|
| **CoinGecko** | 시가총액, 가격, 거래량, 상장 거래소 수 등 | API 무료 티어 있음. 시세·거래량 보조 검증용 | 업계에서 널리 참조 |
| **CoinMarketCap** | 시가총액, 가격, 거래량, 랭킹 | API 유료 비중 있음. 수동 참고 또는 제한적 자동 | 데이터 품질·신뢰도 높음 |
| **업비트 공식** | 원화 기준 시세, 거래량, API | 이미 연동. 기준 데이터 소스 | 국내 현물 기준 |

→ **자동화**: CoinGecko API로 “업비트 외 전 세계 거래량·가격” 보조 비교용으로 사용 가능(예: 3대 거래소 참조와 연계).

### 2.2 온체인·거래소 지표 (숫자 기반, 신뢰도 높음)

| 사이트 | 제공 정보 | 연동 방식 제안 | 비고 |
|--------|-----------|----------------|------|
| **CryptoQuant** | BTC/ETH 등 거래소 유입·유출, 미네어 포지션, 파생물 지표 | API 무료 티어. 설계서 폴백·보조용 | 온체인·거래소 결합 |
| **Glassnode** | 온체인 지표(활성 주소, SOPR, 거래소 잔고 등) | 무료는 제한적. 요약치만 쓸 경우 참고 | 업계 표준에 가까움 |
| **Santiment** | 온체인 + 소셜 감성, 개발 활동 등 | API 있음. 감성·이상 징후 보조 | 온체인+감성 결합 |

→ **자동화**: CryptoQuant는 설계서에서 이미 BTC 온체인 폴백으로 명시. Glassnode/Santiment는 무료 한도 내에서 “보조 점수”나 알림용으로만 쓸 수 있음.

### 2.3 감성·시장 심리 (공포/탐욕 등)

| 사이트 | 제공 정보 | 연동 방식 제안 | 비고 |
|--------|-----------|----------------|------|
| **Alternative.me (Fear & Greed Index)** | 공포/탐욕 지수 0~100 | 설계서 2.1 8개 지표에 이미 포함. 1시간 주기 | 키 불필요, 무료 |
| **Reddit / X (Twitter)** | 코인별 언급량·감성 | 설계서 3.3 멀티 팩터 “커뮤니티 감성” 참조. API 한도 있음 | 최적화 전략 적용 |

→ **자동화**: F&G는 현재 설계대로 사용; Reddit/X는 한도 내에서 감성 점수 보조.

### 2.4 리서치·뉴스·펀더멘털 (수동 참고 위주)

| 사이트 | 제공 정보 | 연동 방식 제안 | 비고 |
|--------|-----------|----------------|------|
| **Messari** | 리서치, 메트릭, 정리된 온체인·재무 지표 | 수동 참고. 이벤트·업데이트 확인용 | 신뢰도 높음 |
| **CoinDesk / CoinTelegraph** | 뉴스, 시장 요약 | 수동 참고. 급등·급락 원인 파악 | 매체 |
| **공식 프로젝트 (예: Bitcoin.org, Ethereum.org)** | 백서, 업그레이드 일정 | 수동 참고. 구조적 이벤트 대비 | 신뢰도 최상 |

→ **자동화**: 뉴스/리서치는 키워드 알림(예: Telegram) 정도만 두고, “진입/비중 로직”에는 직접 넣지 않고 **운영자 판단용**으로만 쓰는 것을 권장.

### 2.5 국내·한국어 정보 (보조 참고)

| 사이트 | 제공 정보 | 연동 방식 제안 | 비고 |
|--------|-----------|----------------|------|
| **업비트 공식 공지/지원** | 유의종목, 공지, API 상태 | 수동·반자동. 유의종목은 설계서 진입 금지 조건에 이미 반영 | 필수 |
| **빗썸·코인원 등 공식** | 공지, 상장/하락 폐지 | 수동 참고. 시장 전반 이슈 확인 | 국내 거래소 |
| **정리된 뉴스/커뮤니티** | 요약 뉴스, 감성 | 수동 참고. 자동 점수화는 선택 | 출처 신뢰도 확인 필요 |

→ **자동화**: “유의종목·투자경보”만 자동 반영(업비트 API 등); 나머지는 수동 참고.

### 2.6 참조 시 원칙

- **1차 판단**: 우리 로직 + 업비트 시세 + 거래소(3대) + 온체인(대표 코인).
- **정보 사이트**:  
  - API로 가져올 수 있는 것은 **보조 점수·필터·알림**으로만 사용하고,  
  - 수동 참고용(리서치, 뉴스)은 **진입/청산 자동 실행에는 직접 넣지 않고**, 운영자가 “이상 징후·원인” 확인할 때만 참조.
- **신뢰도**: 공식(업비트, Etherscan, XRPL, Solscan)·업계 표준(CoinGecko, CryptoQuant, F&G)을 우선하고, 서드파티 요약은 출처를 명시해 두고 사용.
