## 1. 서버 URL 설정

### 1.1 목표
- 배포 시 사용자가 자신의 백엔드 주소(예: `http://192.168.0.10:8000`)를 입력해 앱이 해당 서버에 연결하도록 한다.
- 기본값은 `http://127.0.0.1:8000` (기존과 동일). 저장값이 없으면 기본값 사용.

### 1.2 Flutter

| 구분 | 내용 |
|------|------|
| **저장소** | `shared_preferences` 사용. 키 예: `api_base_url`. |
| **설정 화면** | "연결" 또는 "서버" 섹션 추가. "API 서버 주소" 입력 필드(TextField), placeholder `http://127.0.0.1:8000`, 저장 버튼 또는 onSubmitted 시 저장. 유효성: `http://` 또는 `https://` 로 시작, 끝에 `/` 제거 후 저장. |
| **ApiService** | 생성 시 또는 첫 API 호출 전에 저장된 URL 읽기. `baseUrl`을 상수 대신 **저장된 값 또는 기본값**으로 설정. Dio 인스턴스의 baseUrl을 런타임에 바꿀 수 있도록 하거나, ApiService가 baseUrl getter를 제공하고 요청 시마다 사용. 권장: `ApiService` 생성 시 `SharedPreferences`에서 URL 읽어 `baseUrl` 설정; URL 변경 시 ApiService 재생성 또는 Dio baseUrl 갱신. |
| **구현 요점** | (1) Provider/Repository에서 baseUrl 읽기 함수 제공. (2) 설정 화면에서 URL 저장 후 앱 전역에서 새 URL 사용하도록 갱신(예: ApiService가 baseUrl을 함수로 조회). (3) 잘못된 URL 입력 시 저장만 하고, 실제 요청 시 연결 실패하면 기존 getApiErrorMessage로 안내. |

### 1.3 검증
- 설정에서 URL 변경·저장 후 대시보드 등에서 해당 서버로 요청 나가는지 확인.
- URL 비우기 또는 기본값으로 복구 시 127.0.0.1:8000으로 동작하는지 확인.
- One UI: 입력 필드·카드 스타일 유지.

## 3. 계정 잠금 안내 (남은 시간 표시)

### 3.1 목표
- 로그인 5회 실패 시 15분 잠금. 사용자에게 "O분 후 다시 시도해 주세요"처럼 **남은 시간(분)** 을 알려준다.

### 3.2 백엔드

| 구분 | 내용 |
|------|------|
| **API 변경** | 로그인 403 (계정 잠금) 시 기존: `detail="계정 잠금. 잠시 후 다시 시도하세요."` 유지하고, **응답 본문에 잠금 해제까지 남은 시간(분) 포함**. 방식 1: `detail` 문자열에 "15분 후" 등 포함. 방식 2(권장): 403 시 JSON 본문에 `{"detail": "계정이 일시 잠금되었습니다.", "remaining_minutes": 12}` 형태로 추가 필드 반환. |
| **구현** | `auth.py` 로그인 엔드포인트에서 `user.locked_until and user.locked_until > datetime.utcnow()` 일 때 `remaining_minutes = max(0, ceil((user.locked_until - datetime.utcnow()).total_seconds() / 60))` 계산 후, 403 HTTPException 대신 **JSONResponse(status_code=403, content={"detail": "...", "remaining_minutes": remaining_minutes})** 반환하거나, FastAPI에서 403 시 커스텀 응답 모델 사용. |

### 3.3 Flutter

| 구분 | 내용 |
|------|------|
| **로그인 실패 처리** | 403 응답 시 `response.data`에 `remaining_minutes`가 있으면 해당 값을 사용, 없으면 기본 문구 "계정이 일시 잠금되었습니다. 15분 후 다시 시도해 주세요." 표시. |
| **표시 문구** | "계정이 일시 잠금되었습니다. {remaining_minutes}분 후 다시 시도해 주세요." (한글, SnackBar 또는 로그인 화면 에러 영역) |

### 3.4 검증
- 백엔드: 5회 실패 후 로그인 시 403 + remaining_minutes 반환되는지 확인.
- Flutter: 403 수신 시 남은 분 수가 반영된 문구가 표시되는지 확인.

## 4. 실시간 시세 (REST 기반 현재가·주기 갱신)

### 4.1 목표
- 업비트 현재가(Ticker) API를 활용해 앱에서 "실시간에 가까운" 시세를 표시. WebSocket은 별도 기획으로 두고, 본 항목은 **REST + 주기 갱신**으로 구현.

### 4.2 백엔드

| 구분 | 내용 |
|------|------|
| **API** | `GET /api/v1/market/ticker?markets=KRW-BTC,KRW-ETH` (또는 쿼리 파라미터로 마켓 목록). 인증: **선택**(비인증 허용 시 업비트 공개 API를 백엔드가 프록시; 인증 필수 시 사용자 API 키로 업비트 호출). 공개 시세이므로 **인증 없이** 백엔드에서 업비트 `GET https://api.upbit.com/v1/ticker?markets=...` 호출 후 그대로 반환해도 됨. Rate Limit: 업비트 10회/초 내에서 처리. |
| **구현** | (1) 새 라우터 `app/routers/market.py` 또는 기존 라우터에 `GET /api/v1/market/ticker` 추가. (2) 쿼리: `markets: str` (쉼표 구분, 예: KRW-BTC,KRW-ETH). (3) 업비트 `UpbitClient`는 인증 필요 — ticker는 공개 API이므로 **httpx로 직접** `GET https://api.upbit.com/v1/ticker?markets=...` 호출 후 JSON 반환. (4) 마켓 개수 제한(예: 최대 20개) 적용 권장. |
| **응답 형식** | 업비트 ticker 응답 배열 그대로 반환. 예: `[{ "market": "KRW-BTC", "trade_price": 50000000, "change": "RISE", "signed_change_rate": 0.01, ... }, ...]`. |

### 4.3 Flutter

| 구분 | 내용 |
|------|------|
| **API 호출** | `api_service.dart`에 `getTicker(List<String> markets)` 추가. `GET /api/v1/market/ticker?markets=KRW-BTC,KRW-ETH` 호출, List<Map> 반환. |
| **표시 위치** | (1) **대시보드**: "주요 시세" 또는 "현재가" 카드. 기본 마켓 3~5개(예: KRW-BTC, KRW-ETH, KRW-XRP) 현재가·전일대비(%) 표시. (2) **포지션 화면**: 보유 포지션이 있으면 해당 마켓(예: KRW-BTC) 현재가를 포지션 리스트 옆 또는 상단에 표시. 보유 없으면 대시보드와 동일하게 기본 마켓 시세만 표시. |
| **갱신** | 대시보드/포지션 진입 시 1회 조회 + **30초~60초 주기** 자동 갱신(Timer 또는 Future.delayed 루프). 화면 이탈 시 타이머 취소. 당겨서 새로고침 시에도 최신 ticker 재요청. |
| **UI** | One UI 카드. 코인명, 현재가(원), 전일대비(%·상승 녹색/하락 빨간색). 에러 시 "—" 또는 "조회 실패", getApiErrorMessage로 SnackBar. |

### 4.4 검증
- 백엔드: /market/ticker 호출 시 업비트와 동일한 형식으로 데이터 반환하는지, 마켓 제한 동작 확인.
- Flutter: 대시보드·포지션에서 시세 카드 노출, 30초(또는 60초)마다 갱신, 에러 시 크래시 없이 처리되는지 확인.
