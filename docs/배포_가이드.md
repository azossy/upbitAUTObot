## 1. 필수 환경변수 (.env)

백엔드 `backend/` 디렉터리의 `.env` 파일에 아래 항목을 반드시 설정하세요.

| 변수명 | 설명 | 예시/비고 |
|--------|------|-----------|
| **JWT_SECRET_KEY** | JWT 서명용 비밀키 (64자 hex) | `python -c "import secrets; print(secrets.token_hex(32))"` 로 생성 |
| **ENCRYPTION_KEY** | API 키 암호화용 키 (64자 hex) | 위와 동일한 방식으로 생성 후 저장 |
| **DEBUG** | 개발/운영 구분 | **운영: `false`**, 개발: `true` |
| **CORS_ORIGINS** | 허용 출처 (쉼표 구분) | 운영 시 Flutter 앱 실제 도메인만 입력 (예: `https://myapp.com`) |
| **TELEGRAM_BOT_TOKEN** | 텔레그램 봇 토큰 (선택) | 푸시 알림 사용 시 설정 |
| **TELEGRAM_DEFAULT_CHAT_ID** | 기본 채팅 ID (선택) | 푸시 알림 사용 시 설정 |
| **GOOGLE_APPLICATION_CREDENTIALS** | FCM 서비스 계정 JSON 경로 (선택) | 앱 푸시 알림 사용 시 설정 |
| **DATABASE_URL** | DB 연결 문자열 | 기본: `sqlite+aiosqlite:///./baejjangi.db` |

**기존 upbit* 사용 서버**: 예전에 `upbit_trading.db`, `upbit-backend` 서비스를 쓰던 경우, `scripts/jetson_reinstall_fresh.py` 재설치 시 서비스 파일은 `baejjangi-backend.service`로 자동 이전됩니다. DB는 기존 파일을 그대로 두고 `.env`의 `DATABASE_URL`만 `baejjangi.db`로 바꾼 뒤 `upbit_trading.db` 파일을 `baejjangi.db`로 이름만 바꿔도 됩니다.

- **운영 배포 시**: `JWT_SECRET_KEY`, `ENCRYPTION_KEY` 는 반드시 예시 값이 아닌 새로 생성한 값으로 변경하세요.
- **DEBUG=false** 로 두면 CORS는 `CORS_ORIGINS` 만 허용되고, 500 오류 상세 메시지가 숨겨집니다.

## 2. 개발(DEBUG=true) vs 운영(DEBUG=false) 구분

| 구분 | DEBUG=true (개발) | DEBUG=false (운영) |
|------|------------------|---------------------|
| CORS | 모든 출처 허용 (`*`) | `CORS_ORIGINS` 에 적힌 출처만 허용 |
| 500 오류 | 상세 예외 메시지 노출 | "서버 오류가 발생했습니다" 등 통일 메시지 |
| 용도 | 로컬·Flutter 웹 디버깅 | 실제 서비스 배포 |

운영 환경에서는 반드시 **DEBUG=false** 로 설정하고, **CORS_ORIGINS** 에 Flutter 앱(웹/모바일)에서 접속하는 도메인만 넣어 두세요.

## 3. Jetson/PC에서 상시 실행

### 3.1 systemd 유닛 (Linux / Jetson)

백엔드를 서버 재부팅 후에도 자동으로 띄우려면 systemd 서비스를 사용할 수 있습니다.

1. 작업 디렉터리 예: `/home/user/baejjangi` (서버 설치 시 기본 폴더명 **baejjangi** 권장)
2. 가상환경: `/home/user/baejjangi/backend/venv`
3. 아래 내용으로 `/etc/systemd/system/baejjangi-backend.service` 생성:

```ini
[Unit]
Description=Upbit Trading Backend API
After=network.target

[Service]
Type=simple
User=user
WorkingDirectory=/home/user/baejjangi/backend
EnvironmentFile=/home/user/baejjangi/backend/.env
ExecStart=/home/user/baejjangi/backend/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

4. 적용 및 실행:

```bash
sudo systemctl daemon-reload
sudo systemctl enable baejjangi-backend
sudo systemctl start baejjangi-backend
sudo systemctl status baejjangi-backend
```

- `User`, `WorkingDirectory`, `EnvironmentFile` 경로는 실제 환경에 맞게 수정하세요.
- `.env` 에 **DEBUG=false**, **CORS_ORIGINS** (운영 도메인) 설정이 되어 있어야 합니다.

### 3.2 Docker 실행 예시

`backend/` 에서 Docker로 실행하는 예시입니다.

1. `backend/` 에 `Dockerfile` 이 있다면:

```bash
cd backend
docker build -t baejjangi-backend .
docker run -d --name baejjangi-backend -p 8000:8000 --env-file .env baejjangi-backend
```

2. `.env` 파일은 호스트에 두고 `--env-file .env` 로 넘기거나, `-e` 로 필요한 변수만 넘겨도 됩니다.
3. 운영 시 `.env` 에 **DEBUG=false**, **CORS_ORIGINS**, **JWT_SECRET_KEY**, **ENCRYPTION_KEY** 가 반드시 설정되어 있어야 합니다.
