## 1. 빠른시작

**※ Jetson Tailscale IP (앱 API 서버 주소)**  
`100.80.178.45` → 앱 설정에서 `http://100.80.178.45:8000` 입력.

**※ Jetson SSH**  
`ssh upbit@100.80.178.45`

### 1.1 PC(Windows)에서 로컬 서버 (가장 빠름)

```powershell
cd baejjangi\backend
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
```

**.env**에서 `JWT_SECRET_KEY`, `ENCRYPTION_KEY`를 `python -c "import secrets; print(secrets.token_hex(32))"` 로 생성해 넣기.  
실행: `uvicorn main:app --host 0.0.0.0 --port 8000`  
앱 설정 → API 서버 주소 → `http://PC의IP:8000`

### 1.2 Jetson + Tailscale (요약)

1. Jetson: `curl -fsSL https://tailscale.com/install.sh | sh` → `sudo tailscale up` → 로그인.
2. Jetson IP: `tailscale ip -4` (예: 100.80.178.45).
3. PC에 Tailscale 설치, 같은 계정 로그인.
4. PC에서 `ssh upbit@100.80.178.45`.
5. Jetson에서: `git clone https://github.com/azossy/upbitAUTObot.git baejjangi` → `cd baejjangi/backend` → venv, `pip install -r requirements.txt`, `.env` 설정.
6. systemd 서비스 등록 후 `sudo systemctl enable --now baejjangi-backend`.

앱 API 서버 주소 → `http://100.80.178.45:8000`

상세는 §2 참고.

### 1.3 이메일 인증(SMTP)·문제 해결

회원가입 인증 메일 사용 시 `.env`에 `SMTP_HOST`, `SMTP_USER`, `SMTP_PASSWORD`, `EMAIL_FROM`, `VERIFICATION_CODE_EXPIRE_MINUTES` 설정. (Outlook 예: `SMTP_HOST=smtp-mail.outlook.com`, `SMTP_PORT=587`.)  
**서버가 안 뜰 때**: Jetson에서 `curl -s http://127.0.0.1:8000/health` 무응답이면 `config.py`가 `.env` 변수(SMTP 등)를 인식하는지 확인. Pydantic "Extra inputs" 오류 시 `backend/app/config.py`에 해당 변수 정의 추가 후 재시작.

## 2. Jetson Orin Nano + Tailscale

Jetson에 Tailscale 설치 후 SSH로 접속해 백엔드를 설치합니다.

### 2.1 Jetson에 Tailscale 설치

```bash
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up
```

브라우저에서 로그인. IP 확인: `tailscale ip -4` (100.x.x.x).

### 2.2 PC에서 Tailscale 설치

https://tailscale.com/download 에서 Windows 설치, 같은 계정 로그인.  
SSH: `ssh upbit@100.80.178.45`

### 2.3 Jetson에 백엔드 설치

```bash
cd ~
git clone https://github.com/azossy/upbitAUTObot.git baejjangi
cd baejjangi/backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
nano .env   # JWT_SECRET_KEY, ENCRYPTION_KEY, DEBUG=false, CORS_ORIGINS
```

테스트: `uvicorn main:app --host 0.0.0.0 --port 8000` → PC에서 `http://100.80.178.45:8000/health` 확인.

### 2.4 systemd 상시 실행

`/etc/systemd/system/baejjangi-backend.service` 생성:

```ini
[Unit]
Description=배짱이 Backend API
After=network-online.target

[Service]
Type=simple
User=upbit
WorkingDirectory=/home/upbit/baejjangi/backend
EnvironmentFile=/home/upbit/baejjangi/backend/.env
ExecStart=/home/upbit/baejjangi/backend/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable baejjangi-backend
sudo systemctl start baejjangi-backend
```

## 3. 환경변수·배포

### 3.1 필수 환경변수 (.env)

**구글/카카오 로그인 (선택)**  
- **GOOGLE_CLIENT_ID**: Google Cloud Console에서 발급한 **웹 애플리케이션** OAuth 2.0 클라이언트 ID. 앱의 `OAuthConfig.googleWebClientId`와 동일한 값 사용(공식 권장: 서버 검증용 Web Client ID로 id_token 발급·검증).  
- **KAKAO_REST_API_KEY**: Kakao Developers 앱의 REST API 키. 앱에는 네이티브 앱 키를 `OAuthConfig.kakaoNativeAppKey`로 설정.

| 변수 | 설명 |
|------|------|
| JWT_SECRET_KEY | 64자 hex. `secrets.token_hex(32)` 로 생성 |
| ENCRYPTION_KEY | 64자 hex. 동일 방식 |
| DEBUG | 운영: `false` |
| CORS_ORIGINS | 허용 출처. 운영 시 앱 도메인만 |
| DATABASE_URL | 기본: `sqlite+aiosqlite:///./baejjangi.db` |

선택: SMTP_*, TELEGRAM_*, GOOGLE_APPLICATION_CREDENTIALS 등.

### 3.2 점검 체크리스트 (배포 전)

- [ ] JWT_SECRET_KEY, ENCRYPTION_KEY 새로 생성
- [ ] DEBUG=false
- [ ] CORS_ORIGINS 설정
- [ ] systemd 서비스 정상 기동
- [ ] `curl http://127.0.0.1:8000/health` → 200

## 4. Jetson 점검 체크리스트

Jetson 서버에서 전반 준비가 되었는지 확인할 때 사용합니다.

### 4.1 확인 명령

```bash
sudo systemctl status baejjangi-backend   # active (running)
curl -s http://127.0.0.1:8000/health    # {"status":"ok",...}
curl -s -X POST http://127.0.0.1:8000/api/v1/auth/send-verification-email -H "Content-Type: application/json" -d '{"email":"test@example.com"}'  # 200 또는 503
sudo journalctl -u baejjangi-backend -n 50 --no-pager  # 에러 없음
tailscale status   # up, 100.x.x.x
```
