# 에러 대응 가이드 (배짱이 v1.x)

**목적**: 상용 프로그램 수준의 에러 대응 정리. 개발·검증 시 체크리스트로 사용.

---

## 1. 네트워크·API 에러

| 유형 | 대응 | 구현 위치 |
|------|------|-----------|
| 연결 실패/타임아웃 | getApiErrorMessage 한글 메시지 + "다시 시도" 버튼 | api_service.dart getApiErrorMessage, dashboard/trades/positions 에러 카드에 TextButton "다시 시도" |
| 401 | "인증이 필요합니다. 다시 로그인해 주세요." → 로그인 화면 유도 | getApiErrorMessage 401 반환, 미인증 시 redirect /login (app.dart) |
| 403 | detail + remaining_minutes(계정 잠금) 파싱, 한글 안내 | auth_provider login 403 처리 |
| 5xx | "서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요." + 재시도 가능 UI | getApiErrorMessage, 당겨서 새로고침(RefreshIndicator) 또는 "다시 시도" 버튼 |
| 서버 주소 미설정/잘못됨 | 설정 화면 "연결" 안내 문구 | settings_screen API 서버 주소 helperText: "연결이 안 되면 주소와 백엔드 실행 여부를 확인하세요." |

---

## 2. 입력·검증 에러

| 유형 | 대응 | 구현 위치 |
|------|------|-----------|
| 로그인/회원가입 필드 | 필드별 검증(이메일 형식, 비밀번호 8자, 닉네임), 서버 422 시 detail 파싱해 화면 표시 | login_screen, register_screen, auth_provider |
| 설정(손절/익절 %) | 0~100 범위 밖 입력 시 저장 전 검증, SnackBar "손절 %·익절 %는 0~100 사이로 입력하세요." | settings_screen _saveConfig() |
| API 422 (입력값 오류) | response.data.detail 한글 메시지 → getApiErrorMessage로 SnackBar/화면 표시 | api_service getApiErrorMessage (detail 파싱) |

---

## 3. 화면·상태 에러

| 유형 | 대응 | 구현 위치 |
|------|------|-----------|
| 로딩 중 | CircularProgressIndicator 또는 스플래시, 빈 화면 방지 | 각 화면 _loading 시 Center(child: CircularProgressIndicator()), main.dart _SplashGate |
| 데이터 없음 | "내역이 없습니다", "보유 포지션이 없습니다" 등 빈 상태 메시지 | trades_screen, positions_screen, dashboard 등 |
| 위젯 빌드 실패 | ErrorWidget.builder 로 크래시 대신 안내 문구 | main.dart ErrorWidget.builder |
| 라우터 child null | _FallbackScaffold "로딩 중…" 표시 | app.dart MaterialApp.router builder |

---

## 4. 초기화·외부 연동 에러

| 유형 | 대응 | 구현 위치 |
|------|------|-----------|
| Kakao SDK 초기화 실패 | try/catch 로 앱 기동 유지, 해당 기능만 비활성화 | main.dart KakaoSdk.init in try/catch |
| Firebase/FCM 미설정 | Firebase.apps.isEmpty 시 딥링크 구독 생략, getToken null, 알림 탭 시 크래시 없이 대시보드 이동 | app.dart _setupFcmDeepLink, notification_service |
| FCM 초기화 예외 | NotificationService.initialize().catchError | main.dart |
