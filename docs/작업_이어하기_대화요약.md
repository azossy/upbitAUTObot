## 4. 다음에 켜서 할 일 (우선순위)

1. **GitHub Actions / Release 확인**
   - https://github.com/azossy/upbitAUTObot/actions  
     → v1.0.2용 **Build Release APK** 실행이 성공(녹색)인지 확인.
   - https://github.com/azossy/upbitAUTObot/releases/tag/v1.0.2  
     → **app-release.apk**가 올라와 있으면 다운로드 가능.
   - **실패**면: Actions에서 해당 run → **Build APK (release)** 단계 로그 열어 에러 메시지 확인 후, 그 내용 기준으로 워크플로나 `android/app/build.gradle` 수정.

2. **로컬에서 APK 빌드가 필요할 때**
   - 프로젝트를 **영문 경로**로 복사한 뒤 빌드 (예: `C:\dev\upbitAUTObot\upbit_trading_app`).
   - 또는 `upbit_trading_app\build_apk.bat` 실행 (Flutter 경로 `C:\flutter\bin\flutter.bat` 확인).
   - 빌드 성공 후 수동 업로드:
     ```bash
     gh release upload v1.0.2 upbit_trading_app/build/app/outputs/flutter-apk/app-release.apk --repo azossy/upbitAUTObot --clobber
     ```

3. **문서·스크립트 위치**
   - APK 빌드/배포: `docs/APK_빌드_및_배포.md`
   - 서버(Jetson + Tailscale): `docs/서버_설치_Jetson_Tailscale.md`
   - 트레이딩 로직: `docs/트레이딩_로직_상세_가이드.md`
   - 업로드 스크립트: `upbit_trading_app/upload_apk_to_github.ps1` (태그 인자로 버전 지정)
