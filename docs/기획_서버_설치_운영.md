## 1. 배경 — 실제 서버 세팅에서 얻은 교훈

- **환경 변수**: 백엔드는 `backend/app/config.py`의 Pydantic Settings로 `.env`를 읽음. `.env`에 있지만 Settings에 정의되지 않은 변수가 있으면 (기본 설정 시) "Extra inputs are not permitted"로 기동 실패. → **config에 `extra = "ignore"` 적용**으로 구버전 배포 시에도 기동 가능.
- **필수 설정**: JWT_SECRET_KEY, ENCRYPTION_KEY, (선택) CORS_ORIGINS, OAuth, 텔레그램, FCM, **이메일(SMTP)** 등. 이메일 미설정 시 회원가입 인증 메일은 건너뜀.
- **설치 흐름**: venv 생성 → `pip install -r requirements.txt` → `.env` 생성/수정 → (Jetson 등) systemd 유닛으로 상시 실행. health 확인: `curl -s http://127.0.0.1:8000/health` → `{"status":"ok","version":"1.1.0"}`.
- **사용자 경험**: 수동으로 `.env`를 복사·수정하는 과정에서 실수(누락, 오타)가 나올 수 있으므로, **문답식 설치 스크립트**와 **운영 중 설정 변경용 CLI**로 한 번에 처리할 수 있게 함.

## 2. 설치 스크립트 (install)

### 2.1 목표

- 한 번의 실행으로 **문답식(Q&A)** 으로 필요한 값을 입력받아 `backend/.env`를 생성·갱신하고, venv·의존성 설치, (선택) systemd 안내까지 진행.
- **기존 `.env`가 있는 경우**: 현재 설정값을 보여주고(비밀·토큰은 마스킹), **그대로 사용** 또는 **새로 입력(수정)** 중 선택하게 함.

### 2.2 실행 방법 (안)

- `./scripts/install.sh` (Bash) 또는 `python scripts/install.py` (Python). 프로젝트 루트 또는 backend 기준 경로 명시.

### 2.3 수집 항목 (문답식)

| 구분 | 변수/항목 | 설명 |
|------|------------|------|
| 필수 | JWT_SECRET_KEY | 64자 hex 등. 미입력 시 `secrets.token_hex(32)` 로 생성 제안 |
| 필수 | ENCRYPTION_KEY | 64자 hex. API 키 암호화용 |
| 서버 | CORS_ORIGINS | 쉼표 구분. 기본값 제안 가능 |
| OAuth | GOOGLE_CLIENT_ID, KAKAO_REST_API_KEY | 구글·카카오 로그인용. 없으면 빈 값 허용 |
| 텔레그램 | TELEGRAM_BOT_TOKEN, TELEGRAM_DEFAULT_CHAT_ID | 알림 봇. 없으면 빈 값 |
| FCM | GOOGLE_APPLICATION_CREDENTIALS | 서비스 계정 JSON 경로. 없으면 빈 값 |
| 이메일 | SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASSWORD, EMAIL_FROM, VERIFICATION_CODE_EXPIRE_MINUTES | 회원가입 인증 메일. Outlook 예: smtp-mail.outlook.com, 587 |
| DB | DATABASE_URL | 기본 sqlite+aiosqlite 경로 제안 |
| 기타 | DEBUG | true/false. 운영 시 false 권장 |

입력 시 기본값을 괄호에 표시해 두고 Enter만 치면 기본값 사용하도록 하면 사용성이 좋음.

### 2.4 기존 .env가 있을 때 플로우

1. `backend/.env` 존재 여부 확인.
2. **있음** → 파일에서 키=값 파싱(주석·빈 줄 제외). 비밀·토큰 등 민감 항목은 표시 시 마스킹(예: 앞 2자 + `****`).
3. 현재 설정 요약을 출력하고 선택지 제시:
   - **[U] 그대로 사용**: 기존 .env 유지, 다음 단계(venv, pip install, systemd 안내 등)만 진행.
   - **[M] 새로 입력(수정)**: 문답식으로 항목별 입력받아 .env 갱신 후 다음 단계 진행.
4. **없음** → 문답식으로 전체 입력받아 .env 생성 후 venv·pip·(선택) systemd 안내.

### 2.5 설치 스크립트 후속 단계

- venv 없으면 생성, `pip install -r requirements.txt` 실행.
- (선택) systemd 사용 시: 유닛 파일 예시 경로 또는 내용 안내. 예: [서버_설치_Jetson_Tailscale.md](서버_설치_Jetson_Tailscale.md) 4단계 참고.
- 마지막에 `curl -s http://127.0.0.1:8000/health` 로 확인하라고 안내.

## 3. 운영용 CLI (baejjangi)

### 3.1 목표

- 프로그램명: **baejjangi**. 서버 운영 중 이메일·텔레그램 등 설정을 **한 방에** 변경할 수 있는 CLI.
- **개발 시 `--help` 필수**: 모든 서브커맨드와 옵션에 대해 `baejjangi --help`, `baejjangi set --help`, `baejjangi set telegram --help` 등으로 도움말이 나오도록 구현.

### 3.2 실행 예시

- `baejjangi` 또는 `python -m app.cli` (패키지 구조에 따라 결정).
- `baejjangi --help` → 전체 사용법 및 서브커맨드 목록.
- `baejjangi set --help` → set 서브커맨드 설명.
- `baejjangi set telegram` (또는 `-set -telegram`) → 텔레그램 봇 토큰·기본 Chat ID 문답 후 `backend/.env` 반영.
- `baejjangi set email` (또는 `-set -email`) → SMTP 호스트·포트·사용자·비밀번호·발신자(EMAIL_FROM)·인증 유효분(VERIFICATION_CODE_EXPIRE_MINUTES) 문답 후 `.env` 반영.

### 3.3 서브커맨드 명세

| 서브커맨드 | 설명 | 수집/갱신 항목 |
|------------|------|----------------|
| `set telegram` | 텔레그램 알림 설정 | TELEGRAM_BOT_TOKEN, TELEGRAM_DEFAULT_CHAT_ID |
| `set email` | 이메일(SMTP) 설정 | SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASSWORD, EMAIL_FROM, VERIFICATION_CODE_EXPIRE_MINUTES |
| (선택) `set jwt` | JWT·암호화 키 | JWT_SECRET_KEY, ENCRYPTION_KEY |
| (선택) `set cors` | CORS | CORS_ORIGINS |
| (선택) `set fcm` | FCM 경로 | GOOGLE_APPLICATION_CREDENTIALS |
| (선택) `health` 또는 `status` | 서버 상태 확인 | 로컬에서 GET /health 호출 결과 출력 |

### 3.4 .env 경로

- 기본: `backend/.env` (스크립트/CLI 실행 위치를 프로젝트 루트 또는 backend로 가정).
- (선택) `--env-file /path/to/.env` 옵션으로 경로 지정 가능.

### 3.5 --help 요구사항

- `baejjangi --help`: 프로그램 설명, 사용 가능한 서브커맨드 목록, 공통 옵션(예: --env-file).
- `baejjangi set --help`: set에서 변경 가능한 항목(telegram, email, …) 목록.
- `baejjangi set telegram --help`, `baejjangi set email --help`: 해당 설정 시 입력하는 항목 간단 안내.
