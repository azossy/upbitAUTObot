# 배짱이 v1.0 — 푸시 알림 설정 (FCM + 텔레그램)

**저작자**: 차리 (challychoi@me.com)

매수/매도/손절/긴급정지 시 **텔레그램**과 **FCM 앱 푸시**로 동시 알림을 받을 수 있습니다.

---

## 1. 텔레그램 설정

### 1.1 봇 생성

1. 텔레그램에서 [@BotFather](https://t.me/BotFather) 검색
2. `/newbot` 입력 후 봇 이름·사용자명 설정
3. 발급된 **봇 토큰** 복사

### 1.2 채팅 ID 확인

1. 봇과 대화 시작 (아무 메시지 전송)
2. `https://api.telegram.org/bot<토큰>/getUpdates` 접속
3. `"chat":{"id":123456789` 에서 **chat_id** 확인

### 1.3 백엔드 설정

`backend/.env`:

```
TELEGRAM_BOT_TOKEN=123456:ABC-DEF...
TELEGRAM_DEFAULT_CHAT_ID=123456789
```

### 1.4 앱 설정 화면

설정 → **텔레그램 채팅 ID** 입력 후 저장. (사용자별 `telegram_chat_id`가 DB에 저장됨)

---

## 2. FCM (앱 푸시) 설정

### 2.1 Firebase 프로젝트 생성

1. [Firebase Console](https://console.firebase.google.com/) 접속
2. 프로젝트 생성
3. Android 앱 추가 (패키지: `com.upbit.upbit_trading_app`)
4. `google-services.json` 다운로드 → `upbit_trading_app/android/app/` 에 저장

### 2.2 FlutterFire CLI

```bash
dart pub global activate flutterfire_cli
cd upbit_trading_app
flutterfire configure
```

`firebase_options.dart`가 자동 생성됩니다. 기존 `lib/firebase_options.dart` 플레이스홀더가 교체됩니다.

### 2.3 백엔드 (Firebase Admin)

1. Firebase Console → 프로젝트 설정 → 서비스 계정
2. **새 비공개 키 생성** → JSON 다운로드
3. `backend/firebase-service-account.json` 등으로 저장
4. `backend/.env`:

```
GOOGLE_APPLICATION_CREDENTIALS=./firebase-service-account.json
```

### 2.4 앱 동작

- 로그인 시 FCM 토큰이 자동으로 서버에 등록됩니다.
- 봇 시작/정지, 매수/매도/손절/긴급정지 시 **텔레그램 + FCM** 동시 발송
- 알림 탭 시 앱이 열리고 대시보드로 이동

## 3. 알림 유형

| 유형 | 설명 |
|------|------|
| PUSH-01 | 매수 체결 (코인, 가격, 투자금) |
| PUSH-02 | 매도 체결 (코인, 가격, 손익) |
| PUSH-03 | 손절 (코인, 손실 금액) |
| PUSH-04 | 긴급 정지 (사유) |
| 봇 시작 | 업비트 연동 확인, KRW 잔고 |
| 봇 정지 | 봇이 정지됨 |

---

## 4. 주의사항

- **텔레그램만** 사용 시: `TELEGRAM_BOT_TOKEN`, `TELEGRAM_DEFAULT_CHAT_ID`만 설정
- **FCM만** 사용 시: Firebase 설정 + `GOOGLE_APPLICATION_CREDENTIALS` 설정
- 둘 다 설정 시 **동시 발송**됩니다.
- `telegram_chat_id` 또는 `fcm_token`이 없으면 해당 채널로는 발송하지 않습니다.
