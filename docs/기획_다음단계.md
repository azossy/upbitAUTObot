## 2. 현재까지 개발 완료 범위 (PRD 대비)

- **인증**: 회원가입, 로그인, 로그아웃 (JWT), 비밀번호 변경
- **봇 제어**: 시작/정지(트레이딩 엔진 기동·종료), 상태 조회, API 키 등록·삭제
- **대시보드**: 수익 현황, 수익률 막대·일별 PnL 차트, 보유 포지션, **잔고 표시(KRW)** , 하단 탭
- **설정**: 매매 설정(투자비율·손절/익절·최대포지션), 텔레그램 Chat ID 저장·유지
- **거래내역**: **기간 필터(7/30/90일)** , 목록 조회
- **푸시**: 텔레그램 알림(매수/매도/손절/긴급정지)
- **UI**: One UI 테마, 로그인/회원가입/설정/비밀번호 변경 화면 정리

### 3.3 상세 기획서 (2차: 매매 검증 · 푸시 · 일별 PnL)

**문서**: [docs/기획_매매검증_푸시_일별PnL.md](기획_매매검증_푸시_일별PnL.md)

- **실제 매매 로직 검증**: 현재 상태(봇 시작 시 엔진 미기동), 구현 방향(엔진 기동 방식·검증 모드), 검증 체크리스트.
- **푸시 알림**: 텔레그램 1차(발송 시점·알림 유형·환경변수), FCM 2차, 검증 체크리스트.
- **일별 수익 시계열 API**: API 스펙(pnl-history), Trade 집계, Flutter 차트 확장, 검증 체크리스트.
- **123 작업 지시 초안**이 동 문서 5장에 있으므로, 개발관에게 지시할 때 해당 초안을 복사·수정해 123(화이트보드) 「작업 지시」에 붙여 넣으면 됨.

### 3.4 진입·매각 다중 확인 로직 (상용 수준)

**문서**: [docs/트레이딩_로직_상세_가이드.md](트레이딩_로직_상세_가이드.md)

- **현물 전용**: 숏 없음 → 롱 진입·청산만. 진입/매각을 **2~3중 확인** 후 실행해 오진·과매매 완화.
- **진입 3중**: 1차 시장 국면(상승장·슬롯) → 2차 코인 신호(EMA 골든크로스·4h 정배열·ADX·거래량) → 3차 눌림목 또는 강한 추세 확인.
- **매각 2~3중**: 국면 전환/손절/데드크로스는 각각 2중(연속·유지) 권장; 시간 손절은 3중(12시간+수익률 구간+국면/ADX).
- 실제 전략 코드 구현 시 위 문서의 단계별 조건·체크리스트를 반영할 것.
- **종목 선정·비중·거래소 참조**: [종목선정_비중_거래소참조.md](종목선정_비중_거래소참조.md) — 어떤 종목을 얼마나 넣을지(변동성 반영), 세계 3대 거래소 가격·거래량 참조 방식 정리.
- **온체인·정보 사이트 참조**: [온체인_정보사이트_참조.md](온체인_정보사이트_참조.md) — 대표 대형코인(BTC/ETH/XRP/SOL) 온체인(거래량·유입유출·큰 자금 이동), 신뢰할 만한 코인 정보 사이트 목록·활용 방식.

### 3.5 작업 지시 초안 (123 화이트보드용 — 요약)

코딩 에이전트에게 넣을 지시가 필요할 때 **상세 지시문**은 [기획_매매검증_푸시_일별PnL.md](기획_매매검증_푸시_일별PnL.md) 5장을 사용하고, 요약만 필요할 때 예시는 아래와 같음.

```
[예시]
- (1) 실제 매매 검증: 봇 시작 시 트레이딩 엔진 기동, 업비트 API 호출 검증. 상세: 기획_매매검증_푸시_일별PnL.md 1장.
- (2) 푸시(텔레그램): 매수/매도/손절/긴급정지 시 user.telegram_chat_id로 발송. 상세: 동 문서 2장.
- (3) 일별 PnL API: GET /api/v1/bot/pnl-history?days=30, Flutter 차트 확장. 상세: 동 문서 3장.
```
