## 1. PC(Windows)에서 로컬 서버 띄우기 (가장 빠름)

```powershell
cd upbitAUTObot\backend
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
```

**.env** 열어서 아래만 꼭 수정:

- **JWT_SECRET_KEY**, **ENCRYPTION_KEY**:  
  `python -c "import secrets; print(secrets.token_hex(32))"` 를 두 번 실행해서 나온 64자리 값을 각각 붙여넣기.

저장 후:

```powershell
uvicorn main:app --host 0.0.0.0 --port 8000
```

브라우저에서 `http://127.0.0.1:8000/health` → `{"status":"ok","version":"1.1.0"}` 나오면 성공.

**앱에서**: 설정 → API 서버 주소 → `http://PC의IP:8000` (같은 Wi‑Fi면 PC IP, 예: `http://192.168.0.10:8000`).

## 2. Jetson Orin Nano (Ubuntu) + Tailscale

1. **Jetson**에 Tailscale 설치:  
   `curl -fsSL https://tailscale.com/install.sh | sh` → `sudo tailscale up` → 브라우저에서 로그인.
2. **Jetson Tailscale IP 확인**: `tailscale ip -4` (예: 100.101.102.103).
3. **PC**에 Tailscale 설치 (https://tailscale.com/download), 같은 계정 로그인.
4. **PC에서 Jetson SSH**: `ssh upbit@100.80.178.45`
5. **Jetson에서 백엔드 설치**:
   ```bash
   cd ~
   git clone https://github.com/azossy/upbitAUTObot.git
   cd upbitAUTObot/backend
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   cp .env.example .env
   nano .env   # JWT_SECRET_KEY, ENCRYPTION_KEY 수정
   ```
6. **상시 실행 (systemd)**  
   [서버_설치_Jetson_Tailscale.md](서버_설치_Jetson_Tailscale.md) 4단계 참고해 서비스 파일 생성 후:
   ```bash
   sudo systemctl enable upbit-backend
   sudo systemctl start upbit-backend
   ```

**앱에서**: API 서버 주소 → `http://100.80.178.45:8000` (본인 Jetson Tailscale IP).

## 3. 이메일 인증(회원가입) 사용 시

회원가입 시 인증 메일을 쓰려면 **.env**에 SMTP 설정을 넣어야 합니다.

**challychoi@live.co.kr (Outlook/Live) — 아래 블록 통째로 복사 후 .env에 붙여넣기, 비밀번호만 수정:**

```env
SMTP_HOST=smtp-mail.outlook.com
SMTP_PORT=587
SMTP_USER=challychoi@live.co.kr
SMTP_PASSWORD=여기에_계정비밀번호_또는_앱비밀번호
EMAIL_FROM=배짱이 <challychoi@live.co.kr>
VERIFICATION_CODE_EXPIRE_MINUTES=10
```

- Outlook/Live는 **587 포트(TLS)** 사용. 2단계 인증 켜져 있으면 Microsoft 계정에서 **앱 비밀번호** 발급 후 `SMTP_PASSWORD`에 넣으면 됨.
- 위 6줄을 `.env` 파일 맨 아래에 붙여넣고, `SMTP_PASSWORD`만 실제 값으로 바꾸면 됨.

| 변수명 | 설명 |
|--------|------|
| SMTP_HOST | SMTP 서버 (Outlook: smtp-mail.outlook.com) |
| SMTP_PORT | 587 (TLS) 또는 465 (SSL) |
| SMTP_USER | 이메일 주소 (예: challychoi@live.co.kr) |
| SMTP_PASSWORD | 계정 비밀번호 또는 앱 비밀번호 |
| EMAIL_FROM | 발신 표시 (예: 배짱이 \<challychoi@live.co.kr\>) |
| VERIFICATION_CODE_EXPIRE_MINUTES | 인증 코드 유효 분 (기본 10) |

설정하지 않으면 인증 메일 발송이 건너뛰어지고, 테스트용으로는 인증 없이 가입될 수 있습니다.

## 서버가 안 뜰 때 (Jetson: curl /health 무응답·연결 거부)

**증상**: `curl -s http://127.0.0.1:8000/health` 아무 응답 없음 또는 "연결이 거부됨". `systemctl status upbit-backend` → exit-code 실패.

**원인**: Jetson의 `backend/app/config.py`가 예전 버전이면 `.env`에 있는 변수(SMTP, 이메일 등)를 인식하지 못해 Pydantic이 "Extra inputs are not permitted"로 기동 직후 종료함.

**해결 (둘 중 하나)**

### 방법 1: Jetson에서 config.py를 최신과 동일하게 맞추기

1. Jetson SSH 접속 후:
   ```bash
   nano ~/upbitAUTObot/backend/app/config.py
   ```
2. **`GOOGLE_APPLICATION_CREDENTIALS: str = ""` 다음**에 아래 블록이 **반드시 있는지** 확인. 없으면 추가:
   ```python
       # 이메일 발송 (회원가입 인증·축하 메일)
       SMTP_HOST: str = ""
       SMTP_PORT: int = 587
       SMTP_USER: str = ""
       SMTP_PASSWORD: str = ""
       EMAIL_FROM: str = "배짱이 <noreply@example.com>"
       VERIFICATION_CODE_EXPIRE_MINUTES: int = 10
   ```
3. **`class Config:` 안**에 아래 한 줄이 있는지 확인. 없으면 추가:
   ```python
       extra = "ignore"  # .env에만 있고 여기 없는 변수는 무시
   ```
4. 저장(Ctrl+O, Enter) 후 종료(Ctrl+X). 재시작:
   ```bash
   sudo systemctl restart upbit-backend
   curl -s http://127.0.0.1:8000/health
   ```
   → `{"status":"ok", ...}` 나오면 성공.

### 방법 2: PC에서 코드 푸시 후 Jetson에서 git pull

1. **PC(프로젝트 폴더)** 에서 최신 config.py가 반영된 상태로 커밋·푸시:
   ```powershell
   git add backend/app/config.py
   git commit -m "config: 이메일 Settings + extra ignore"
   git push
   ```
2. **Jetson** 에서:
   ```bash
   cd ~/upbitAUTObot && git pull
   sudo systemctl restart upbit-backend
   curl -s http://127.0.0.1:8000/health
   ```

**여전히 실패할 때**: Jetson에서 `sudo journalctl -u upbit-backend -n 30 --no-pager` 로 에러 메시지 확인. 다른 ValidationError나 import 오류가 있으면 해당 내용 기준으로 추가 수정.
