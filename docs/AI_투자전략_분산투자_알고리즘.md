# AI 메뉴 투자 전략 및 분산 투자 알고리즘

## 1. 개요

- **종목 지정**: 오토(개미엔진 자동 선정) / 수동(업비트 원화 종목 중 최대 10개 사용자 선택).
- **투자 전략**: 수익성 우선 / 손실 최소 / 균형 / 개미엔진 판단 중 하나 선택.
- 선택한 1~10종목에 대해 **총 투자금을 전략에 따라 비율 배분**한다.

## 2. 투자 전략별 알고리즘

| 전략 (API 값) | 설명 | 배분 방식 |
|---------------|------|-----------|
| **수익성 우선** (`profit_first`) | 상승 기대치가 큰 종목에 더 투자 | 점수^1.5 비율 (상위 종목 편중) |
| **손실 최소** (`loss_min`) | 리스크 분산 우선 | 균등 배분 (1/n) |
| **균형** (`balanced`) | 수익·리스크 균형 | 균등 배분 (1/n) |
| **개미엔진 판단** (`engine_decision`) | 엔진 점수 그대로 반영 | 점수 비율 그대로 배분 |

### 2.1 수익성 우선 (profit_first)

- 각 종목의 **상승 가능성 점수**(엔진/시장 점수)를 \(s\)라 할 때, 가중치 \(w = (s + 0.1)^{1.5}\).
- 총 투자금을 \(w\) 비율로 배분.
- 효과: 점수가 높은 종목에 더 많이 투자되어 수익 극대화에 유리.

### 2.2 손실 최소 (loss_min) / 균형 (balanced)

- 선택된 종목 수 \(n\)에 대해 **균등 배분**: 종목당 `총투자금 / n`.
- 효과: 한 종목 급락 시 타 종목으로 분산.

### 2.3 개미엔진 판단 (engine_decision)

- 엔진이 반환한 종목별 점수를 그대로 비율로 사용.
- `allocate_krw_by_scores(total_krw, [(market, score), ...])`와 동일.

## 3. 구현 위치

- **백엔드**: `backend/app/trading/engine.py`
  - `allocate_by_strategy(total_krw, market_scores, strategy, min_per_market=5000)` 사용.
  - 봇 설정 `Bot.config["allocation_strategy"]`에서 전략 읽음.
- **API**: `PUT /api/v1/bot/config`에 `allocation_strategy` 필드 (profit_first | loss_min | balanced | engine_decision).
- **앱**: AI 메뉴에서 종목 지정(오토/수동) 아래 **투자 전략** 4가지 중 하나 선택 후 저장.

## 4. 적용 순서 (다종목 매수 시)

1. 종목 목록 확정: 오토면 엔진이 선정, 수동이면 사용자가 선택한 최대 10종목.
2. 각 종목에 대해 엔진/시장 점수 수집 (예: `market_score` 또는 엔진 시그널 강도).
3. `allocate_by_strategy(총투자금, [(market, score), ...], config["allocation_strategy"])` 호출.
4. 반환된 `{ market: krw_amount }`대로 종목별 매수 금액 실행.

## 5. 포트폴리오 구성 재기획 및 API

- **재기획서**: `docs/포트폴리오_구성_재기획.md` — 종목 선정 → 전략 선택 → 점수 수집 → 배분 계산 → 실행 흐름 정리.
- **예상 배분 API**: `GET /api/v1/bot/portfolio-preview` — 현재 봇 설정(종목·투자 전략) 기준 예상 배분. 수익성우선/손실최소/균형/개미엔진판단 반영.
- **앱**: AI 메뉴에서 저장된 종목·전략 기준으로 "예상 배분" 카드에 총 투자 금액 및 종목별 금액·비중 표시.
