| 번호 | 항목 | 확인 내용 |
|------|------|-----------|
| G-1 | **기획서·PRD 정합성** | docs/기획_다음단계.md·PRD와 현재 구현 범위(1~5차)가 일치하는지 확인. 미구현 필수 항목이 있으면 123.txt "기획관 지시"에 명시. |
| G-2 | **문서 최신화** | 기획_다음단계.md의 "현재까지 개발 완료 범위"에 5차(서버 URL·디스클레이머·계정잠금·실시간시세) 반영. v1.0 제외 범위(iOS·출금·백오피스) 유지. |
| G-3 | **123.txt 지시 정리** | "기획관 → 개발관·검증관 지시·안내"에 v1.0 최종 점검 후 다음 액션(재작업 반영·배포 준비 등)이 명확한지 확인. |
| G-4 | **UI/UX 요구사항** | docs/UI_UX_가이드_적용.md와 PRD 비기능(One UI·접근성)이 작업 지시·검증 체크리스트에 반영되어 있는지 확인. |
| G-5 | **배포·운영 문서** | docs/배포_가이드.md에 필수 환경변수·실행 방법·체크리스트가 출시에 충분한지 검토. |

**완료 시**: 123.txt "기획 업데이트(기획관)" 또는 "기획관 지시" 섹션에 "v1.0 최종 점검 완료" 및 갱신한 문서 목록 기재.

## 2. [개발관] 최종 점검 항목

| 번호 | 항목 | 확인 내용 |
|------|------|-----------|
| D-1 | **작업지시서 미해결 항목** | "코드 검증 결과 및 작업지시서"의 "■ 작업지시서"에 번호로 된 미해결 수정 요청이 있으면 해당 항목 순서대로 수정·보완 코딩 후 "최근 작업 내역"에 기록. |
| D-2 | **5차 개발 완료 여부** | 서버 URL 설정·디스클레이머·계정 잠금(remaining_minutes)·실시간 시세(ticker) 4가지가 모두 구현되었는지 확인. 미완 시 docs/기획_v1.0_출시_4가지_작업.md 기준으로 완료. |
| D-3 | **코드·설정 일관성** | 앱 baseUrl이 저장된 URL/기본값 사용, 로그인 403 시 remaining_minutes 파싱, 대시보드/설정에 디스클레이머 문구, ticker API·주기 갱신 동작. 린트/빌드 에러 없음. |
| D-4 | **예외·메시지** | API 실패·네트워크 오류 시 getApiErrorMessage 등 한글 메시지, 크래시 없이 SnackBar/화면 메시지 처리. |
| D-5 | **배포 전 코드** | .env.example·주석에 비밀/키 하드코딩 없음. 버전 표기(앱 1.0.0, /health version) 유지. |

**완료 시**: 123.txt "최근 작업 내역"에 "v1.0 최종 점검 — 작업지시서 반영·5차 확인·일관성 점검" 요약 기재. 수정한 파일 목록 기재.

## 4. 실행 순서 (권장)

1. **[기획관]** 123.txt 확인 → 본 문서 1장(G-1~G-5) 체크 → 기획서·문서 갱신 → 123.txt 기획관 섹션에 "v1.0 최종 점검 완료" 기재.
2. **[개발관]** 123.txt 확인 → 본 문서 2장(D-1~D-5) 체크 → 작업지시서 미해결 항목 수정·5차 미완 시 완료·일관성 점검 → "최근 작업 내역" 갱신. 필요 시 "검증관 할당 작업"에 "v1.0 최종 점검 검증 요청" 기재.
3. **[검증관]** 123.txt 확인 → "v1.0 최종 점검 검증 요청" 또는 기획관 지시에 따라 본 문서 3장(V-1~V-7) 체크 → "코드 검증 결과 및 작업지시서"에 v1.0 최종 점검 결과 블록 작성 → 할당 [완료!], 작업 완료 현황 갱신. 재작업 항목 있으면 작업지시서에 기재.

이후 작업지시서에 항목이 있으면 개발관이 수정 후 검증관이 재점검 가능.
