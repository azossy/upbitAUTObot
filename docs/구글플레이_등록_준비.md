# 구글플레이 등록 준비 — OAuth(구글·카카오 로그인) 설정

**용도**: 구글플레이에 앱을 등록할 때 필요한 **OAuth(구글/카카오 로그인)** 설정 안내.  
사용자가 「구글플레이 등록용」이라고 하면 이 문서를 기준으로 OAuth 설정을 안내한다.

---

## 1. 왜 이때 설정하나

- OAuth는 **구글플레이 등록 시점**에 맞춰 설정하기로 함.
- 설정 전에는 앱에서 구글/카카오 버튼 탭 시 *"이 기기에서는 구글/카카오 로그인이 설정되지 않았습니다. 이메일 로그인 또는 회원가입을 이용해 주세요."* 만 표시됨.

---

## 2. 구글 로그인 설정

| 단계 | 내용 |
|------|------|
| 1 | [Google Cloud Console](https://console.cloud.google.com/) → 프로젝트 선택 또는 생성 |
| 2 | **API 및 서비스** → **사용자 인증 정보** → **OAuth 2.0 클라이언트 ID** 생성 |
| 3 | **Android** 클라이언트: 패키지명 `com.upbit.upbit_trading_app`, SHA-1 등록 |
| 4 | **웹 애플리케이션** 클라이언트 생성 → **웹 클라이언트 ID** 복사 |
| 5 | **앱**: `upbit_trading_app/lib/config/oauth_config.dart` 의 `googleWebClientId` 에 위 웹 클라이언트 ID 입력 |
| 6 | **백엔드**: 서버 `.env` 에 `GOOGLE_CLIENT_ID=위와_동일한_웹_클라이언트_ID` 추가 (Jetson 등 원격은 `scripts/jetson_set_oauth_env.py` 활용 가능) |

---

## 3. 카카오 로그인 설정

| 단계 | 내용 |
|------|------|
| 1 | [Kakao Developers](https://developers.kakao.com/) → 앱 생성/선택 |
| 2 | **앱 키**에서 **네이티브 앱 키** 확인 |
| 3 | **카카오 로그인** 활성화, **동의 항목**에서 필요 시 이메일·프로필 설정 |
| 4 | **앱**: `upbit_trading_app/lib/config/oauth_config.dart` 의 `kakaoNativeAppKey` 에 네이티브 앱 키 입력 |
| 5 | **백엔드**: 서버 `.env` 에 `KAKAO_REST_API_KEY=REST_API_키` 추가 (Jetson 등은 `jetson_set_oauth_env.py` 활용) |

---

## 4. 설정 후 확인

- 앱 재빌드 후 **구글 계정으로 로그인** / **카카오 계정으로 로그인** 동작 확인.
- 미가입 계정 → 회원가입 화면 → 닉네임 입력 → 가입 완료 후 홈 이동 플로우 확인.

자세한 플로우·에러 처리 기준은 **docs/테스트_실행_결과_보고.md** §5·§6 참고.
