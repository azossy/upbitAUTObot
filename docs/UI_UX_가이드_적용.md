## 3. 컴포넌트 (One UI 컴포넌트)

| 컴포넌트 | 기준 | 적용 요약 |
|----------|------|-----------|
| **앱바** | 뒤로가기·제목·액션(아이콘 우선, 최대 3개) | 제목 중앙, 액션은 아이콘 버튼 |
| **하단 탭** | 아이콘+텍스트, 4~5개, 스크롤 시에도 고정 | NavigationBar 사용, 탭 명으로 화면 제목 구성 |
| **버튼** | 플랫(툴바/다이얼로그) vs 컨테이너드(강조) 혼용 금지 | 한 화면 내 한 종류. 본문 강조는 FilledButton |
| **다이얼로그** | 사용자 액션 필요 시 하단에서 올라오기 | BottomSheet 또는 하단 정렬 모달 권장 |
| **리스트** | 제목 한 줄, 부제목 제한 없음 | ListTile title/subtitle 활용 |
| **프로그레스** | 전체 가리지 않고 제한 영역에 표시 | 로딩 시 앱바·버튼 영역 인디케이터 권장 |

## 4. 화면별 체크리스트 (개발관 참고)

- **로그인/회원가입**: 보는 영역(타이틀) 위, 입력·버튼 아래. 포커스 블록(둥근 카드) 사용.  
- **대시보드**: 수치·차트는 포커스 블록, 중요 액션(시작/정지)은 컨테이너드 버튼. 하단 탭 고정.  
- **포지션/거래내역**: 리스트 제목 한 줄, 카드 radius 20dp, 빈 상태 시 아이콘+문구.  
- **설정**: 섹션 제목 → 카드/리스트. Slider·입력 필드 One UI 스타일 유지.  
- **새로 추가하는 화면**: 좌우 24dp, 카드 20dp, Primary 색상·다크 모드 테마 유지.
