# 트레이딩 엔진 로직 시뮬레이션 테스트

**목적**: 기획서(진입·매각 다중확인)대로 트레이딩 로직이 동작하는지, **가상 데이터**로 검증합니다. 실제 업비트 API는 호출하지 않습니다.

## 실행 방법

```bash
cd backend
python scripts/test_trading_logic.py
```

- **기본**: 80봉 가상 1시간봉 데이터로 진입 1·2·3차, 매각(손절/익절) 조건을 단계별 판단해 결과를 출력합니다.
- **간단 시나리오(데모)**  
  ```bash
  python scripts/test_trading_logic.py --simple
  ```  
  - T30에서 **강제 매수**, 이후 가격 하락 시 **손절(-2.5%)** 한 번 발생하도록 설계된 시나리오입니다.
  - 매수 1회 → 매도(손절) 1회가 나오는지 확인해, **매각 로직(손절)**이 기획대로 동작하는지 판단할 수 있습니다.

## 결과 해석

- **[요약]**: 매수 신호·매도 신호가 몇 회, 어느 시점(tick/타임스탬프)에서 났는지 표시됩니다.
- **[상세 판단]**: 매수/매도가 발생한 봉에 대해  
  - 시장국면, 1차/2차/3차 통과 여부  
  - 골든크로스, RSI≤50, 단기 EMA 근처 여부  
  - 포지션, **결정(매수 / 매도(손절·익절))**  
  을 출력합니다.

결과값을 보면서 다음을 확인하면 됩니다.

1. **진입**: 1차(시장 상승·슬롯) → 2차(골든크로스·정배열·ADX·거래량) → 3차(눌림목 등)가 기획서와 맞게 적용되는지  
2. **매각**: 손절(-2.5%)·익절(+5%)·국면 하락·데드크로스 등 **우선순위**와 **2단계 확인**이 반영되는지
