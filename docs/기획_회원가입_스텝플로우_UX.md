# 배짱이 — 회원가입 스텝 플로우 및 비밀번호 표시 UX 기획

**저작자**: 차리 (challychoi@me.com)  
**참조**: docs/UI_UX_가이드_적용.md, One UI 디자인 가이드  
**목적**: 상용 프로그램 수준의 친절한 안내와 스텝 바이 스텝 회원가입 플로우 정립. 개발관 구현 시 본 문서와 123(화이트보드) 「작업 지시」를 함께 참고.

---

## 1. 공통 UX 원칙

- **상용 프로그램**: 모든 과정에서 친절한 안내 문구, 스텝 바이 스텝 진행.
- **UI/UX**: docs/UI_UX_가이드_적용.md 준수 — 보는 영역(상단) / 인터랙션(하단), 좌우 24dp, 카드 radius 20dp, Primary #0381FE, FilledButton 일관.
- **비밀번호 입력란**: 로그인·회원가입·비밀번호 변경 등 **모든 비밀번호 필드**에 **눈 아이콘(visibility)** 을 두어, 탭 시 비밀번호 표시/숨김 전환.

---

## 2. 비밀번호 필드 — 눈 아이콘 (표시/숨김)

| 적용 화면 | 내용 |
|-----------|------|
| 로그인 | 비밀번호 입력란 오른쪽에 눈 아이콘. 탭 시 obscureText 토글. |
| 회원가입 | 비밀번호 입력란 오른쪽에 눈 아이콘. (스텝 2 화면) |
| 비밀번호 변경 | 현재 비밀번호·새 비밀번호·확인 비밀번호 각각 눈 아이콘. (이미 있을 수 있음 — 확인 후 통일) |

- **아이콘**: 숨김 상태 → `Icons.visibility_off_outlined`, 표시 상태 → `Icons.visibility_outlined`. 또는 Material 디자인 기준 동일 의미 아이콘.
- **위치**: TextField `suffixIcon`으로 배치. 터치 영역 최소 48dp 유지.

---

## 3. 회원가입 플로우 — 스텝 구성

로그인 화면에서 **「회원가입」** 버튼 탭 시 **새 화면**으로 이동하며, 아래 순서대로 **한 단계씩** 진행한다.

---

### Step 1 — 환영 및 이메일 입력

**화면 구성 (위 → 아래)**

1. **보는 영역 (상단)**  
   - **표제**: `환영합니다!!`  
   - **소개 문구**: 배짱이 앱에 대한 짧은 소개 (예: "업비트 자동매매를 쉽고 안전하게 도와주는 배짱이와 함께하세요.")  
   - **안내 문구**: `이메일 주소를 입력하세요.`

2. **입력 영역 (카드/포커스 블록)**  
   - 이메일 주소 입력 필드 (label: "이메일", prefixIcon: 이메일 아이콘)  
   - **닉네임** 입력 필드 (label: "닉네임", prefixIcon: person 아이콘) — 백엔드 API 필수 항목이므로 본 단계에서 함께 입력받음.

3. **인터랙션 영역 (하단)**  
   - 버튼: **`다음 ->`** (FilledButton, Primary 색상)

**동작**  
- "다음 ->" 탭 시: 이메일 형식·닉네임(1~100자) 검증. 통과 시 Step 2로 전환.

---

### Step 2 — 비밀번호 입력

**화면 구성 (위 → 아래)**

1. **보는 영역 (상단)**  
   - **안내 문구**: `비밀번호를 입력하세요.`  
   - (선택) 부가 안내: "8자 이상 입력해 주세요."

2. **입력 영역 (카드/포커스 블록)**  
   - 비밀번호 입력 필드 (label: "비밀번호 (8자 이상)", prefixIcon: lock, **suffixIcon: 눈 아이콘**으로 표시/숨김)

3. **인터랙션 영역 (하단)**  
   - 버튼: **`다음 ->`**

**동작**  
- "다음 ->" 탭 시: 비밀번호 8자 이상 검증. 통과 시 **서버에 인증 메일 발송** (기존 `sendVerificationEmail(email)` 호출, 이메일·닉네임·비밀번호는 Step 3에서 사용하므로 상태 유지) 후 Step 3로 전환.

---

### Step 3 — 인증 코드 입력 및 가입 완료

**화면 구성 (위 → 아래)**

1. **보는 영역 (상단)**  
   - **안내 문구**: `이메일을 열고 발송된 코드를 입력하세요.`  
   - (선택) "인증 메일이 도착하지 않았다면 스팸함을 확인해 주세요."

2. **입력 영역 (카드/포커스 블록)**  
   - 코드 입력 필드: **숫자 6자리** (keyboardType: number, maxLength: 6, hint: "6자리 코드")  
   - (선택) 이메일 주소 표시: "발송 대상: xxx@xxx.com" (읽기 전용)

3. **인터랙션 영역 (하단)**  
   - 버튼: **`확인`** (인증 코드 확인 및 가입 완료 — 기존 `verifyAndRegister` 호출)

**인증 성공 시 (verifyAndRegister 성공)**  
- **성공 메시지 화면** (같은 화면 내 카드 또는 단일 메시지 영역):  
  - `축하합니다! 인증되었습니다.`  
  - `로그인 화면에서 정식 로그인을 하시면 됩니다.`  
- 버튼: **`확인`**  
- "확인" 탭 시 → **로그인 화면**(`/login`)으로 이동. (기존 `context.go('/login')`)

**인증 실패 시**  
- 기존과 동일: 에러 메시지 표시(한글), 재입력 유도.

## 5. 라우팅·백엔드 연동

- **라우트**: 기존 `/register` 사용. 단, Step 1·2·3은 **한 화면(RegisterScreen)** 내에서 `_step`(1/2/3) 및 성공 상태로 전환.
- **API**:  
  - Step 2 "다음 ->" 시: `sendVerificationEmail(email)` 호출 후 Step 3으로.  
  - Step 3 "확인" 시: `verifyAndRegister(email, password, nickname, code)` 호출.  
- **뒤로가기**: Step 2 → Step 1, Step 3 → Step 2. Step 1에서 뒤로가기 → `/login`.
