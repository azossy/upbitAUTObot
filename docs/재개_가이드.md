## 1. 한 줄 요약

- **진행**: 1~4차 개발 완료, **4차 검증 요청됨**(검증관이 수행 대기).
- **다음에 할 일**: 검증관 채팅에서 **「123 확인해줘」** → 4차 검증 수행. 기획관은 123 확인 후 기획서 갱신·다음 단계(5차 또는 개발 완료 정리) 결정.

## 2. 공용 화이트보드 (반드시 확인)

| 파일 | 설명 |
|------|------|
| **123** (화이트보드, 프로젝트 루트 `123.md`) | 기획관·개발관·검증관이 함께 쓰는 작업 지시·완료 현황·검증 요청·최근 작업 내역. **다음에 할 일은 여기서 확인.** |

- **에이전트 역할**: 기획관(P에이전트), 개발관(코딩 에이전트), 검증관(테스트 에이전트). 각자 123(화이트보드)의 해당 섹션을 읽고 갱신.
- **완료 표기**: 작업/검증 끝나면 해당 블록에 **[완료!] YYYY-MM-DD** 표기, "■ 작업 완료 현황" 테이블 갱신.
- **검증 후 수정**: 검증관이 "■ 작업지시서"에 항목을 적으면 → **개발관**이 123 확인 후 해당 항목대로 수정 코딩.

## 3. 현재 진행 상황 (저장 시점 기준)

| 단계 | 개발 | 검증 |
|------|------|------|
| 1차 | 완료 (상용 1차: 예외처리, 입력검증, 비밀번호 변경, 서버 연결 상태) | 완료 |
| 2차 | 완료 (매매 검증, 푸시 텔레그램, 일별 PnL) | 완료 |
| 3차 | 완료 (거래 기간 필터, 잔고 표시) | 완료 |
| 4차 | 완료 (CSV 내보내기, 배포 가이드) | **요청됨** — 검증관이 123 확인 시 수행 |

- **지금 할 일**: **검증관**이 123(화이트보드) 확인 → 기획 4차 검증 수행 → 검증 결과 갱신 → [완료!]+날짜, "개발 완료 현황" 갱신.

## 4. 다음번에 다시 할 때 체크리스트

### [기획관 / 사용자]
1. **123**(화이트보드, `123.md`) 열기 → "개발 완료 현황", "담당자별 이번에 할 일", "검증관 할당 작업" 확인.
2. **docs/기획_다음단계.md** 갱신 (모니터링 결과, 다음 단계 반영).
3. 4차 검증 완료 여부 확인. 완료 시 → 5차 작업 지시를 넣을지, "개발 완료" 정리할지 결정.

### [개발관]
1. **123**(화이트보드) 열기 → "작업 지시 (개발관)"에서 [진행중] 항목 확인.
2. "■ 작업지시서"(코드 검증 결과 및 작업지시서)에 재작업 항목 있으면 해당 항목대로 수정.
3. 새 작업 지시가 있으면 해당 순서대로 코딩 → "최근 작업 내역" 기록, [완료!], 검증 요청 기재.

### [검증관]
1. **123**(화이트보드) 열기 → "검증관 할당 작업"에 검증 요청 있는지 확인.
2. 요청 있으면 해당 검증 수행 → "코드 검증 결과 및 작업지시서" 갱신 → 할당 블록 [완료!]+날짜, "■ 작업 완료 현황" 갱신.
3. 재작업 필요 시 "■ 작업지시서"에 개발관 전달 항목 번호로 기재.
