## 1. PRD·기획 대비 부족/미완 부분

### 1.1 필수(PRD) 중 미구현 또는 약한 부분

| PRD ID | 항목 | 현재 상태 | 비고 |
|--------|------|-----------|------|
| **UPBIT-04** | 주문 생성/취소 | 트레이딩 엔진이 잔고 조회만 수행, 실제 주문/취소 로직 없음 | v1.0은 “검증용” 루프만 있어 실제 매매는 사용자 책임 안내 필요 |
| **UPBIT-05** | 실시간 시세 (WebSocket) | 미구현 | 캔들/가격 차트 없음. 중기 기획에 있음 |
| **AUTH-05** | 계정 잠금 (5회 실패 15분) | 백엔드만 구현, 앱에서 403 시 메시지만 표시 | 잠금 시 **남은 시간** 표시 없음 → 사용자 혼란 가능 |
| **TRADE-03** | 거래내역 코인별 필터 | 권장 항목, 미구현 | 많은 거래 시 불편 |

### 1.2 인프라·운영 관점

| 항목 | 현재 상태 | 리스크 |
|------|-----------|--------|
| **API 서버 URL** | `api_service.dart`에 `127.0.0.1:8000` 하드코딩 | 실제 배포 시 사용자가 서버 주소를 바꿀 수 없음. 빌드별/환경별 설정 필요 |
| **JWT 갱신** | Refresh Token 발급은 있으나 앱에서 사용 여부 불명 | Access Token 만료(30분) 시 재로그인 요구 가능. 장시간 사용 시 불편 |
| **에러 로그/복구** | 봇 에러 시 로그 저장·설정에서 최근 에러 보기 없음 | 장애 시 사용자가 원인 파악하기 어려움 |

## 2. 사용자 입장에서 있으면 좋은 기능

### 2.1 신뢰·안심 (상용 앱 수준)

| 제안 | 설명 | 우선순위 |
|------|------|----------|
| **이용약관·개인정보처리방침** | 회원가입 또는 설정/My에 링크. 앱 내 웹뷰 또는 외부 URL | 권장 (기획_다음단계 4.3) |
| **자동매매 책임 안내(디스클레이머)** | 최초 봇 시작 전 또는 설정 상단에 “실제 매매는 사용자 책임, 투자 손실에 대해 서비스는 책임지지 않음” 등 한 줄 안내 | 권장 |
| **로그인 실패 잠금 안내** | 403 수신 시 “계정이 잠금되었습니다. O분 후 다시 시도해 주세요.” 처럼 **남은 시간** 표시 (백엔드에서 남은 분 수 내려주면 표시 가능) | 권장 |

### 2.2 사용성

| 제안 | 설명 | 우선순위 |
|------|------|----------|
| **서버 URL 설정** | 설정 화면에서 “API 서버 주소” 입력·저장 (기본값 127.0.0.1:8000). 배포 시 사용자가 자신의 서버 주소 입력 | 높음 (배포 전) |
| **CSV 실제 파일 저장** | 현재 클립보드 복사만 지원. Android에서 `getTradesExportCsv` 결과를 파일로 저장(다운로드 폴더 등) 옵션 | 중간 |
| **당겨서 새로고침 일관** | 대시보드 등 주요 화면에 RefreshIndicator 이미 있음. 거래내역/포지션 등 모든 목록 화면에서 동일하게 유지 | 낮음 (이미 대부분 적용됨) |
| **빈 목록 안내 문구** | 포지션/거래내역 0건일 때 “보유 포지션이 없습니다” 등 문구·아이콘으로 명시 | 낮음 (일부만 있으면 보강) |
| **거래내역 코인별 필터** | TRADE-03. 코인 선택 시 해당 코인만 목록 표시 | 중기 |

### 2.3 모니터링·투명성

| 제안 | 설명 | 우선순위 |
|------|------|----------|
| **봇 상태 문구 구체화** | “실행 중”/“중지” 외에 “마지막 체크: O분 전”, “에러 시 사유 한 줄” 등 | 중간 |
| **설정에 최근 에러 메시지** | 봇/API 에러 시 서버가 저장한 최근 에러 1건을 설정 또는 대시보드에 표시 | 중간 |

### 2.4 접근성·다국어

| 제안 | 설명 | 우선순위 |
|------|------|----------|

## 3. v1.0 출시 전 권장 조치 (우선순위)

### 3.1 출시 직전 권장 (필수에 가까움)

1. **서버 URL 설정**
   - 앱에서 API base URL을 설정에서 입력·저장하고, `ApiService`가 해당 URL 사용하도록 변경.
   - 기본값 `http://127.0.0.1:8000` 유지. 배포 시 사용자가 자신의 서버 주소 입력 가능해야 함.

2. **자동매매 책임 안내(디스클레이머)**
   - 봇 “시작” 버튼 근처 또는 설정 상단에 고정 문구 1줄:  
     “자동매매로 인한 손실은 사용자 책임이며, 서비스는 투자 결과에 대해 책임지지 않습니다.”

3. **계정 잠금 시 안내**
   - 로그인 403 시 `detail`이 “계정 잠금…”이면,  
     “계정이 일시 잠금되었습니다. 15분 후 다시 시도해 주세요.” 등으로 표시.  
     (가능하면 백엔드에서 `remaining_minutes` 등 반환 후 “O분 후 다시 시도” 표시)

### 3.2 있으면 좋음 (v1.0 또는 v1.0.1)

4. **이용약관·개인정보처리방침 링크**
   - 회원가입 화면 하단 또는 설정/My에 “이용약관”, “개인정보처리방침” 링크 (URL만 있어도 됨).

5. **CSV 파일 저장**
   - “CSV 내보내기” 시 클립보드 복사 + (Android) 파일로 저장 옵션.

6. **봇 에러 노출**
   - 봇 상태가 에러일 때 설정 또는 대시보드에 “최근 오류: …” 한 줄 표시 (백엔드에 에러 저장 필드/API 필요).

### 3.3 v1.0 이후로 미룰 수 있는 항목

- JWT Refresh Token 앱 연동 (만료 전 자동 갱신)
- 거래내역 코인별 필터 (TRADE-03)
- 실시간 시세/캔들 차트 (UPBIT-05)
- iOS 지원 (PRD v1 제외 범위)
