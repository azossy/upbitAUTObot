# 포트폴리오 구성 재기획

## 1. 목적

- **종목 선정 완료 후** 총 투자금을 각 종목에 **얼마씩 배분할지** 포트폴리오를 구성한다.
- 사용자가 AI 메뉴에서 선택한 **투자 전략**(수익성 우선 / 손실 최소 / 균형 / 개미엔진 판단)을 반영하여 배분 비율을 결정한다.

## 2. 전제 조건

| 입력 | 설명 |
|------|------|
| 투자 가능 금액 | 원화 잔고 중 봇이 사용할 금액 (또는 사용자 지정 금액) |
| 종목 목록 | 수동: 사용자 선택 최대 10종목 / 오토: 엔진이 선정한 종목 목록 |
| 투자 전략 | `profit_first` / `loss_min` / `balanced` / `engine_decision` |

## 3. 포트폴리오 구성 흐름

```
[종목 선정 완료] → [투자 전략 선택] → [종목별 점수 수집*] → [전략별 배분 계산] → [종목별 투자 금액]
                                 (* engine_decision / profit_first 시 필요)
```

1. **종목 목록 확정**  
   - 수동: `config.selected_markets` (최대 10개)  
   - 오토: 매 루프 시점에 엔진/시장 데이터 기반 선정 (구현 시 엔진 연동)

2. **종목별 점수(상승 가능성)**  
   - `engine_decision` / `profit_first`: 개미엔진 또는 시장 지표 기반 점수 사용 (0~10 등).  
   - 점수 미제공 시(예: 미리보기): 균등 점수(1.0)로 가정 → 전략별로 아래 배분 규칙만 적용.

3. **전략별 배분 규칙** (기존 `allocate_by_strategy`와 동일)

   | 전략 | 배분 방식 | 비고 |
   |------|-----------|------|
   | 수익성 우선 | 가중치 = (점수+0.1)^1.5, 비율 배분 | 상위 종목 편중 |
   | 손실 최소 | 균등 배분 (1/n) | 리스크 분산 |
   | 균형 | 균등 배분 (1/n) | 리스크 분산 |
   | 개미엔진 판단 | 점수 비율 그대로 배분 | 엔진 신호 반영 |

4. **제약**  
   - 종목당 최소 투자금: 5,000원 (업비트 최소 주문 단위 등).  
   - 해당 최소금 미달 종목은 배분 결과에서 제외.

## 4. 출력 형태

- **종목별 투자 금액**: `{ "KRW-BTC": 300000, "KRW-ETH": 200000, ... }`  
- **미리보기/API용**: `[ { "market": "KRW-BTC", "krw": 300000, "weight_pct": 30.0 }, ... ]`  
  - `weight_pct`: 해당 종목 비중(%).

## 5. 적용 위치

| 구분 | 내용 |
|------|------|
| 배분 계산 | `backend/app/trading/engine.py` — `allocate_by_strategy()` |
| 봇 설정 | `Bot.config["allocation_strategy"]`, `selected_markets` |
| 포트폴리오 미리보기 | `GET /api/v1/bot/portfolio-preview` — 전략·종목 반영 예상 배분 |
| 실제 매매 시 | 트레이딩 루프에서 매수 신호 발생 시 `allocate_by_strategy(총금액, 점수목록, config["allocation_strategy"])` 호출 후 종목별 금액으로 주문 |

## 6. 점수 산출 (실전 vs 미리보기)

- **실전**: 개미엔진 `/signal` 또는 시장 지표(예: 최근 수익률)로 종목별 점수 산출 후 `allocate_by_strategy`에 전달.  
- **미리보기(API)**: 엔진/시장 데이터 없이 호출 시 **균등 점수(1.0)** 로 계산하여,  
  - `loss_min` / `balanced`: 균등 배분  
  - `profit_first` / `engine_decision`: 균등 점수이므로 역시 균등 배분으로 표시.  
  - 실제 봇 운영 시에는 엔진 점수가 반영되므로 전략별 차이가 발생.

## 7. 정리

- 사용자 선택 **수익성 우선 / 손실 최소 / 균형 / 개미엔진 판단**은 모두 **포트폴리오 구성(종목별 금액 배분)** 에 반영된다.  
- 배분 로직은 `allocate_by_strategy` 한 곳에서 전략별로 통일되어 있으며,  
  - **미리보기 API**로 설정된 종목·전략 기준 예상 배분을 확인하고,  
  - **실제 매매** 시 동일 전략과 (엔진) 점수를 사용해 포트폴리오를 구성하면 된다.
